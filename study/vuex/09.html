# vuex4 插件

Vuex 4 支持使用插件来扩展其功能。插件是一个函数，接受一个 Vuex store 作为参数，可以用来监听 store 的 mutation，添加新的 action，添加新的 getter，等等。

下面是一个示例插件：

const myPlugin = (store) => {
  // 在每次 mutation 之后打印状态变化
  store.subscribe((mutation, state) => {
    console.log(`mutation ${mutation.type}`, state)
  })

  // 添加一个新的 action，用来将 count 加倍
  store.dispatch('doubleCount', () => {
    store.commit('doubleCount')
  })
}

const store = createStore({
  state() {
    return {
      count: 0
    }
  },
  mutations: {
    increment(state) {
      state.count++
    },
    doubleCount(state) {
      state.count *= 2
    }
  },
  actions: {
    doubleCount({ commit }) {
      commit('doubleCount')
    }
  },
  plugins: [myPlugin]
})

在这个示例中，我们定义了一个插件函数 myPlugin，该函数接受一个 Vuex store 作为参数。我们使用 subscribe() 函数来监听 mutation，当 mutation 发生时，打印出状态变化。我们还使用 dispatch() 函数添加了一个新的 action，该 action 将调用 doubleCount mutation 来将 count 加倍。

最后，我们通过将插件函数传递给 plugins 选项来将插件添加到 Vuex store 中。

需要注意的是，插件函数的执行顺序是按照添加顺序执行的。也就是说，如果有多个插件函数，它们的执行顺序将按照它们添加到 Vuex store 中的顺序执行。因此，如果插件函数之间存在依赖关系，需要特别注意插件的添加顺序。

除了使用函数形式的插件外，Vuex 4 还支持使用对象形式的插件。对象形式的插件需要包含 install() 方法，该方法接受一个 Vuex store 作为参数，用来实现插件的具体功能。

以下是一个示例对象形式的插件：

const myPlugin = {
  install(store) {
    // 在每次 mutation 之后打印状态变化
    store.subscribe((mutation, state) => {
      console.log(`mutation ${mutation.type}`, state)
    })

    // 添加一个新的 action，用来将 count 加倍
    store.dispatch('doubleCount', () => {
      store.commit('doubleCount')
    })
  }
}

const store = createStore({
  state() {
    return {
      count: 0
    }
  },
  mutations: {
    increment(state) {
      state.count++
    },
    doubleCount(state) {
      state.count *= 2
    }
  },
  actions: {
    doubleCount({ commit }) {
      commit('doubleCount')
    }
  },
  plugins: [myPlugin]
})

在这个示例中，我们定义了一个对象形式的插件 myPlugin，该插件包含一个 install() 方法，该方法接受一个 Vuex store 作为参数。我们使用 subscribe() 函数来监听 mutation，当 mutation 发生时，打印出状态变化。我们还使用 dispatch() 函数添加了一个新的 action，该 action 将调用 doubleCount mutation 来将 count 加倍。

最后，我们将插件对象传递给 plugins 选项来将插件添加到 Vuex store 中。

需要注意的是，对象形式的插件必须包含 install() 方法，并且该方法必须在插件被添加到 Vuex store 之前被调用。因此，对象形式的插件不能像函数形式的插件一样直接作为数组元素传递给 plugins 选项。如果要添加多个对象形式的插件，需要将它们分别作为单独的元素添加到数组中。

除了直接将插件添加到 Vuex store 中之外，我们还可以通过 use() 方法来动态安装插件。use() 方法接受一个插件函数或对象作为参数，该方法会检查插件是否已经被安装过，如果没有，就会调用插件的 install() 方法来安装插件。

以下是一个使用 use() 方法动态安装插件的示例：

const myPlugin = {
  install(store) {
    // 在每次 mutation 之后打印状态变化
    store.subscribe((mutation, state) => {
      console.log(`mutation ${mutation.type}`, state)
    })
  }
}

const store = createStore({
  state() {
    return {
      count: 0
    }
  },
  mutations: {
    increment(state) {
      state.count++
    }
  },
  actions: {
    increment({ commit }) {
      commit('increment')
    }
  }
})

// 安装插件
store.use(myPlugin)

在这个示例中，我们定义了一个插件 myPlugin，该插件只会在 mutation 发生时打印状态变化。我们使用 use() 方法将插件动态安装到 Vuex store 中。

需要注意的是，插件是一次性的，也就是说，如果我们使用 use() 方法动态安装插件，那么该插件只会对动态安装之后的 mutation 进行监听。如果我们想要对所有的 mutation 进行监听，应该在创建 Vuex store 时将插件添加到 plugins 选项中。

除了动态安装插件，我们还可以使用 createLogger() 方法创建一个日志插件，该插件会在每次 mutation 发生时打印相关信息，包括 mutation 的名称、触发 mutation 的 action 的名称（如果有的话）以及 mutation 前后的状态快照。

以下是一个使用 createLogger() 方法创建日志插件的示例：

import { createStore, createLogger } from 'vuex'

const store = createStore({
  state() {
    return {
      count: 0
    }
  },
  mutations: {
    increment(state) {
      state.count++
    }
  },
  actions: {
    increment({ commit }) {
      commit('increment')
    }
  },
  plugins: [createLogger()]
})

在这个示例中，我们使用 createLogger() 方法创建了一个日志插件，并将该插件添加到 plugins 选项中。这个日志插件会在每次 mutation 发生时打印相关信息。

需要注意的是，日志插件只应该在开发环境中使用，因为它会打印大量的调试信息，对性能有一定的影响。在生产环境中，我们应该关闭日志插件，可以通过在创建 Vuex store 时设置 devtools 选项来实现。例如：

import { createStore } from 'vuex'

const store = createStore({
  state() {
    return {
      count: 0
    }
  },
  mutations: {
    increment(state) {
      state.count++
    }
  },
  actions: {
    increment({ commit }) {
      commit('increment')
    }
  },
  devtools: process.env.NODE_ENV === 'development'
})

在这个示例中，我们根据当前环境变量来设置 devtools 选项，只有在开发环境中才会启用 Vuex Devtools。

除了使用 createLogger() 方法创建日志插件，我们还可以自定义插件来实现一些特殊的功能。自定义插件需要实现一个方法，该方法会在每次 mutation 发生时被调用，并接收当前的 state 和 mutation 作为参数。例如，以下是一个自定义插件的示例：

const myPlugin = store => {
  // 在每次 mutation 发生时被调用
  store.subscribe((mutation, state) => {
    // 打印当前的 mutation 和 state
    console.log('mutation:', mutation)
    console.log('state:', state)
  })
}

const store = createStore({
  state() {
    return {
      count: 0
    }
  },
  mutations: {
    increment(state) {
      state.count++
    }
  },
  plugins: [myPlugin]
})

在这个示例中，我们定义了一个名为 myPlugin 的插件，该插件在每次 mutation 发生时被调用，并通过 store.subscribe() 方法来订阅 mutation 事件。在插件方法中，我们可以通过参数获取当前的 mutation 和 state，并进行相应的操作，例如打印相关信息。

需要注意的是，自定义插件的执行顺序与它们在 plugins 选项中的顺序相同。因此，如果我们希望插件按照特定的顺序执行，需要注意插件的顺序。

除了在插件方法中处理数据外，我们还可以通过插件来扩展 Vuex 的功能，例如增加一个 reset() 方法来重置 state，或者增加一个 persist() 方法来将 state 持久化到本地存储中。

下面是一个增加 reset() 方法的示例：

const resetPlugin = store => {
  store.reset = () => {
    // 将 state 重置为初始值
    const initState = store._modules.root.state
    store.replaceState(initState)
  }
}

const store = createStore({
  state() {
    return {
      count: 0
    }
  },
  mutations: {
    increment(state) {
      state.count++
    }
  },
  plugins: [resetPlugin]
})

// 使用 reset 方法重置 state
store.reset()

在这个示例中，我们通过 store.reset() 方法来重置 state，该方法内部将 state 替换为初始值。需要注意的是，为了避免与其他插件或 Vuex 自身的方法重名，我们使用了一个名为 resetPlugin 的插件，并将 reset() 方法挂载到了 store 上。

类似地，我们还可以通过插件来实现将 state 持久化到本地存储中的功能。这个功能常用于需要在页面刷新后恢复状态的场景。下面是一个增加 persist() 方法的示例：

const persistPlugin = store => {
  // 从本地存储中恢复 state
  const persistedState = JSON.parse(localStorage.getItem('state'))
  if (persistedState) {
    store.replaceState(persistedState)
  }

  // 在每次 mutation 发生时保存 state 到本地存储中
  store.subscribe((mutation, state) => {
    localStorage.setItem('state', JSON.stringify(state))
  })

  // 增加一个 persist() 方法，用于手动将 state 持久化到本地存储中
  store.persist = () => {
    localStorage.setItem('state', JSON.stringify(store.state))
  }
}

const store = createStore({
  state() {
    return {
      count: 0
    }
  },
  mutations: {
    increment(state) {
      state.count++
    }
  },
  plugins: [persistPlugin]
})

// 使用 persist 方法将 state 持久化到本地存储中
store.persist()

在这个示例中，我们使用了一个名为 persistPlugin 的插件，该插件在初始化时从本地存储中恢复 state，并在每次 mutation 发生时将 state 保存到本地存储中。我们还增加了一个 persist() 方法，用于手动将 state 持久化到本地存储中。

需要注意的是，将 state 持久化到本地存储中可能会带来一些安全问题，因为存储在本地的数据可以被用户篡改。因此，我们需要在实际应用中谨慎使用这个功能，或者采用其他更安全的方式来持久化数据。
