# vuex4 核心概念 state

Vuex 4 是一个基于 Vue.js 的状态管理库。在 Vuex 中，核心的概念之一是状态（state），它是存储在 Vuex 中的数据源。下面我们来详细讲解 Vuex 中的 state。

# 状态（state）

在 Vuex 中，状态（state）是存储在 Vuex 中的数据源，即应用程序中的全局数据。它是一个 JavaScript 对象，包含着应用程序的所有状态数据。当一个组件需要访问状态时，它可以从 Vuex 中获取它们，而不是直接从组件的本地状态访问它们。

在一个 Vue 组件中，可以使用 this.$store.state 访问整个应用程序的状态。以下是一个示例：

// 在组件中访问状态
export default {
  computed: {
    count() {
      return this.$store.state.count
    }
  }
}

在上面的示例中，通过 $store.state.count 获取了 Vuex 中的状态。

在 Vuex 中，状态不能直接被改变，而是通过提交 mutations 来改变状态。这是为了更好地跟踪状态的变化，并可以记录状态变化的历史记录。

以下是一个简单的 Vuex Store 示例，其中包含一个名为 count 的状态属性：

import { createStore } from 'vuex'

const store = createStore({
  state() {
    return {
      count: 0
    }
  }
})

export default store

在上面的示例中，使用 createStore 方法创建了一个名为 store 的 Vuex Store，该 Store 包含了一个名为 count 的状态属性，它的默认值是 0。

在实际应用中，状态通常会包含更多的属性和更复杂的数据结构。通过使用 Vuex，我们可以集中管理这些状态数据，并保证它们的一致性。

总之，状态是 Vuex 中的核心概念之一，它是存储在 Vuex 中的数据源，是整个应用程序的共享数据。通过 mutations 改变状态，使应用程序中的组件可以获得最新的状态数据。

# 在组件中访问状态

在组件中访问 Vuex 中的状态非常简单，可以通过 mapState 辅助函数或 this.$store.state 来访问。下面我们分别介绍这两种方式。

# 使用 mapState 辅助函数

mapState 辅助函数是 Vuex 提供的一个工具函数，它可以帮助我们更方便地访问 Vuex 中的状态。该函数可以接受一个数组或对象作为参数，返回一个计算属性对象，该对象包含了从 Vuex 中获取的状态。

以下是一个使用 mapState 辅助函数访问状态的示例：

import { mapState } from 'vuex'

export default {
  computed: {
    ...mapState(['count'])
  }
}

在上面的示例中，使用 mapState 辅助函数将名为 count 的状态映射到了组件的计算属性中。通过这种方式，可以在模板中直接访问状态，如下所示：

<template>
  <div>{{ count }}</div>
</template>

# 使用 this.$store.state

另一种访问 Vuex 中状态的方式是通过 this.$store.state 来访问。以下是一个示例：

export default {
  computed: {
    count() {
      return this.$store.state.count
    }
  }
}

在上面的示例中，通过 $store.state.count 获取了 Vuex 中的状态。这种方式与 mapState 辅助函数类似，但需要在模板中使用计算属性。

# 简化访问

以上两种方式都可以访问 Vuex 中的状态，但是当状态变得复杂时，访问起来就比较麻烦。为了简化访问，可以使用计算属性来访问嵌套的状态。

以下是一个示例：

export default {
  computed: {
    fullName() {
      return `${this.$store.state.user.firstName} ${this.$store.state.user.lastName}`
    }
  }
}

在上面的示例中，使用计算属性来访问名为 user 的嵌套状态。但是，这样的访问方式非常冗长且难以维护。为了简化访问，可以使用 mapState 辅助函数。

以下是一个使用 mapState 辅助函数访问嵌套状态的示例：

import { mapState } from 'vuex'

export default {
  computed: {
    ...mapState({
      firstName: state => state.user.firstName,
      lastName: state => state.user.lastName
    }),
    fullName() {
      return `${this.firstName} ${this.lastName}`
    }
  }
}

在上面的示例中，使用 mapState 辅助函数将名为 firstName 和 lastName 的嵌套状态映射到了组件的计算属性中。通过这种方式，可以在模板中直接访问状态。

# 修改状态

在 Vuex 中，我们不能直接修改状态，而是需要使用 mutations 来修改状态。这样做的好处是，可以记录下每一次状态的变化，便于跟踪和调试。以下是一个修改状态的示例：

// mutations.js
export default {
  increment(state) {
    state.count++
  },
  decrement(state) {
    state.count--
  }
}

// store.js
import mutations from './mutations'

export default new Vuex.Store({
  state: {
    count: 0
  },
  mutations
})

// component.vue
export default {
  methods: {
    increment() {
      this.$store.commit('increment')
    },
    decrement() {
      this.$store.commit('decrement')
    }
  }
}

在上面的示例中，定义了两个 mutations，分别为 increment 和 decrement，用于增加和减少状态中的计数器。然后在组件中，通过 $store.commit 方法调用 mutations 来修改状态。

需要注意的是，在调用 mutations 时，需要传入一个表示当前状态的参数 state。在 mutations 中，可以通过这个参数来修改状态。

# 状态的响应式

在 Vuex 中，状态是响应式的，即当状态发生变化时，相关的组件会自动更新。这是因为 Vuex 中的状态是使用 Vue.js 的响应式系统来实现的。

在 Vuex 中，当组件访问状态时，实际上是在访问 Vue.js 的响应式对象。当状态发生变化时，Vue.js 会自动检测到变化，并通知相关的组件更新视图。

以下是一个使用 Vuex 修改状态并更新组件的示例：

// store.js
export default new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++
    }
  }
})

// component.vue
export default {
  computed: {
    count() {
      return this.$store.state.count
    }
  },
  methods: {
    increment() {
      this.$store.commit('increment')
    }
  }
}

在上面的示例中，定义了一个名为 count 的状态，并且定义了一个 mutations increment，用于增加计数器的值。在组件中，通过 $store.state.count 访问状态，并在方法中调用 $store.commit 方法来修改状态。由于状态是响应式的，所以当状态发生变化时，相关的组件会自动更新。

# 访问嵌套状态

在实际开发中，我们有时需要访问嵌套在状态对象中的状态。例如，在一个购物车应用程序中，我们可能会有以下状态：

state: {
  cart: {
    items: [],
    shippingAddress: {
      name: '',
      address: '',
      city: '',
      state: '',
      zip: ''
    }
  }
}

在上面的状态中，cart 包含了购物车的所有信息，包括商品列表和运送地址。如果我们想要访问运送地址的信息，可以使用点语法来访问嵌套的状态：

computed: {
  shippingAddress() {
    return this.$store.state.cart.shippingAddress
  }
}

在上面的计算属性中，使用 $store.state.cart.shippingAddress 访问了嵌套在状态对象中的状态。注意，由于状态是响应式的，所以当状态发生变化时，相关的组件会自动更新。

# 对象展开运算符

有时，我们需要在修改状态时同时修改多个属性。例如，在购物车应用程序中，我们可能需要同时修改购物车中的商品列表和运送地址。为了做到这一点，可以使用对象展开运算符：

this.$store.commit('updateCart', {
  items: [...newItems],
  shippingAddress: {...newShippingAddress}
})

在上面的示例中，我们调用了名为 updateCart 的 mutations，并传入了一个对象作为参数。在这个对象中，我们使用了对象展开运算符，将新的商品列表和运送地址分别合并到了状态对象中。

# 总结

我们学习了 Vuex 中的核心概念之一：状态。我们了解了如何在 Vuex 中定义状态、在组件中访问状态以及如何使用 mutations 修改状态。同时，我们也了解了 Vuex 中状态的响应式特性，以及如何使用计算属性来访问嵌套的状态。

在本文中，我们学习了如何在 Vuex 中访问嵌套的状态，并使用点语法访问状态对象中的属性。同时，我们还学习了如何在修改状态时同时修改多个属性，并使用对象展开运算符合并新的状态信息。

Vuex 中的状态是非常强大的，它可以帮助我们更好地组织和管理数据。因此，我们需要仔细考虑状态的设计，并合理使用 Vuex 中提供的各种工具和技巧来优化我们的代码和应用程序的性能。
