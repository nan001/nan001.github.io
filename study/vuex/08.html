# vuex4 组合式API

Vuex 4 引入了新的组合式 API，它可以让我们更灵活地使用 Vuex。在 Vuex 3 中，我们通常是通过对象来定义 State、Mutation、Action 等，但这种方式在大型项目中会变得比较混乱和难以维护。在 Vuex 4 中，我们可以使用组合式 API 来更加灵活地组织代码。

组合式 API 是基于 Vue 3 中的 Composition API 实现的，它允许我们将逻辑代码按功能划分为多个可重用的函数，从而提高代码的可维护性和可读性。在 Vuex 4 中，我们可以使用以下几个 API 来实现组合式开发：

useStore()：用于在组件中获取 Store 实例。 useVuex()：用于在普通 JavaScript 中获取 Store 实例。 useState()：用于在组件中获取 State。 useGetters()：用于在组件中获取 Getter。 useMutations()：用于在组件中获取 Mutation。 useActions()：用于在组件中获取 Action。 下面我们来具体了解一下这些 API。

# useStore()

useStore() 是一个在组件中获取 Store 实例的函数。在 Vue 3 中，我们通常使用 setup() 函数来获取 Store 实例，并将其返回给模板中使用。例如：

<template>
  <div>{{ count }}</div>
</template>

<script>
import { useStore } from 'vuex'

export default {
  setup() {
    const store = useStore()

    return {
      count: computed(() => store.state.count)
    }
  }
}
</script>

在上面的示例中，我们使用 useStore() 函数获取了 Store 实例，并使用 computed() 函数来监听 State 的变化，从而在模板中显示当前的计数值。

# useVuex()

useVuex() 是一个在普通 JavaScript 中获取 Store 实例的函数。如果我们需要在普通 JavaScript 中使用 Vuex，我们可以通过 useVuex() 来获取 Store 实例，从而进行状态管理。例如：

import { useVuex } from 'vuex'

const { store } = useVuex()

console.log(store.state.count)

在上面的示例中，我们使用 useVuex() 函数获取了 Store 实例，并使用 console.log() 函数来输出当前的计数值。

# useState()

useState() 是一个在组件中获取 State 的函数。如果我们需要在组件中获取 State，我们可以通过 useState() 函数来实现。例如：

<template>
  <div>{{ count }}</div>
</template>

<script>
import { useState } from 'vuex'

export default {
  setup() {
    const { count } = useState(['count'])

    return {
      count
    }
  }
}
</script>

在上面的示例中，我们使用 useState() 函数获取了 State，并将其解构为 count 变量。在 return 中，我们将 count 变量返回给模板中使用。

# useGetters()

useGetters() 是一个在组件中获取 Getter 的组合式 API，它的使用方法如下：

import { useGetters } from 'vuex'

export default {
  setup() {
    const { getter1, getter2 } = useGetters(['getter1', 'getter2'])
    // ...
  }
}

useGetters() 接收一个数组作为参数，数组中包含需要获取的 Getter 名称。在组件中，我们可以使用对象解构将返回的 Getter 绑定到组件的变量上，并直接使用这些变量。

另外，useGetters() 也可以接收一个函数作为参数，这个函数的参数为所有 Getter，返回值为需要获取的 Getter。使用方法如下：

import { useGetters } from 'vuex'

export default {
  setup() {
    const { getter1, getter2 } = useGetters((getters) => {
      return {
        getter1: getters.getter1,
        getter2: getters.getter2,
      }
    })
    // ...
  }
}

这种方法可以在获取 Getter 的同时进行一些计算或筛选操作。

# useMutations()

useMutations() 是一个在组件中获取 Mutation 的组合式 API，它的使用方法如下：

import { useMutations } from 'vuex'

export default {
  setup() {
    const { mutation1, mutation2 } = useMutations(['mutation1', 'mutation2'])
    // ...
  }
}

useMutations() 接收一个数组作为参数，数组中包含需要获取的 Mutation 名称。在组件中，我们可以使用对象解构将返回的 Mutation 绑定到组件的变量上，并直接调用这些变量，即可触发对应的 Mutation。

另外，useMutations() 也可以接收一个函数作为参数，这个函数的参数为所有 Mutation，返回值为需要获取的 Mutation。使用方法如下：

import { useMutations } from 'vuex'

export default {
  setup() {
    const { mutation1, mutation2 } = useMutations((mutations) => {
      return {
        mutation1: mutations.mutation1,
        mutation2: mutations.mutation2,
      }
    })
    // ...
  }
}

这种方法可以在获取 Mutation 的同时进行一些计算或筛选操作。

# useActions()

useActions() 是一个在组件中获取 Action 的组合式 API，它的使用方法如下：

import { useActions } from 'vuex'

export default {
  setup() {
    const { action1, action2 } = useActions(['action1', 'action2'])
    // ...
  }
}

useActions() 接收一个数组作为参数，数组中包含需要获取的 Action 名称。在组件中，我们可以使用对象解构将返回的 Action 绑定到组件的变量上，并直接调用这些变量，即可触发对应的 Action。

另外，useActions() 也可以接收一个函数作为参数，这个函数的参数为所有 Action，返回值为需要获取的 Action。使用方法如下：

import { useActions } from 'vuex'

export default {
  setup() {
    const { action1, action2 } = useActions((actions) => {
      return {
        action1: actions.action1,
        action2: actions.action2,
      }
    })
    // ...
  }
}

这种方法可以在获取 Action 的同时进行一些计算或筛选操作。

除了上述两种方法，useActions() 还可以接收一个对象作为参数，这个对象中的键是需要获取的 Action 名称，值是对应的 Action。这种方法可以用于对 Action 进行重命名或者重新组合，例如：

import { useActions } from 'vuex'

export default {
  setup() {
    const { addTodo } = useActions({
      addTodo: 'todos/addTodo',
    })
    // ...
  }
}

上面的例子中，我们将名为 "todos/addTodo" 的 Action 重命名为 "addTodo"，这样在组件中就可以直接调用 "addTodo" 触发 Action 了。useActions() 是一个非常方便的组合式 API，可以大大简化组件中触发 Action 的代码量。

# 当我们使用组合式 API 时，我们可以使用以下函数来创建一个新的 Vuex store：

import { createStore } from 'vuex'

const store = createStore({
  state() {
    return {
      count: 0
    }
  },
  mutations: {
    increment(state) {
      state.count++
    }
  }
})

在这里，我们通过 createStore() 函数创建了一个新的 Vuex store。createStore() 函数接受一个对象作为参数，该对象包含了我们需要在 store 中定义的状态、突变、操作和 getter。

# 在组合式 API 中，我们使用以下函数来定义状态、突变、操作和 getter：

reactive()：创建一个响应式的对象，用来表示状态。 computed()：创建一个计算属性，用来定义 getter。 watchEffect()：创建一个副作用函数，用来定义操作。 watch()：创建一个 watcher，用来监听状态的变化。

下面是一个使用组合式 API 来定义 Vuex store 的例子：

import { createStore, reactive, computed, watchEffect } from 'vuex'

const store = createStore({
  state: reactive({
    count: 0
  }),
  mutations: {
    increment(state) {
      state.count++
    }
  },
  actions: {
    asyncIncrement({ commit }) {
      return new Promise((resolve) => {
        setTimeout(() => {
          commit('increment')
          resolve()
        }, 1000)
      })
    }
  },
  getters: {
    doubledCount: computed((state) => state.count * 2)
  },
  modules: {}
})

watchEffect(() => {
  console.log(store.state.count)
})

store.dispatch('asyncIncrement')

在这个例子中，我们使用 reactive() 函数来创建了一个响应式的状态对象。我们使用 computed() 函数来定义了一个名为 doubledCount 的 getter，该 getter 返回 count 的两倍。我们使用 watchEffect() 函数来创建了一个副作用函数，当 count 改变时，该函数会自动运行。

我们还定义了一个名为 asyncIncrement 的操作，该操作使用 Promise 来模拟异步请求。我们在操作中使用 commit() 函数来提交突变，该突变会将 count 加一。

最后，我们创建了一个新的 Vuex store，包含了我们定义的状态、突变、操作和 getter。我们使用 watchEffect() 函数来监听 count 的变化，并调用 dispatch() 函数来调用 asyncIncrement 操作。
