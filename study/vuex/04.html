# vuex4 核心概念 Mutation

Mutation 是 Vuex 中的一个核心概念，用于同步地修改 store 中的 state。Mutation 可以看作是一个纯函数，它接受 state 作为第一个参数，并接受一个可选的负载（payload）作为第二个参数，通过修改 state 的值来改变 store 中的数据。

Mutation 的定义方式如下：

const store = new Vuex.Store({
  state: {
    count: 0
  },
  mutations: {
    increment(state) {
      state.count++
    },
    incrementBy(state, payload) {
      state.count += payload.amount
    }
  }
})

在上面的代码中，我们定义了两个 mutation：increment 和 incrementBy。increment mutation 没有接受任何参数，它通过 state.count++ 将 state 中的 count 属性加 1。而 incrementBy mutation 接受一个负载参数 payload，其中包含一个 amount 属性，通过 state.count += payload.amount 将 state 中的 count 属性增加了 payload.amount。

我们可以通过 store.commit 方法来触发 mutation：

store.commit('increment')
store.commit('incrementBy', { amount: 10 })

在上面的代码中，我们分别通过 store.commit('increment') 和 store.commit('incrementBy', { amount: 10 }) 来触发 increment 和 incrementBy mutation。commit 方法的第一个参数是 mutation 名称，第二个参数是一个可选的负载对象。

需要注意的是，我们不能直接通过修改 state 的值来改变 store 中的数据，因为这样会破坏 Vuex 的数据流动。所有对 store 中数据的修改都必须通过 mutation 来进行，这样 Vuex 才能正确地追踪数据的变化并通知所有订阅者。

除了直接调用 store.commit 方法，我们还可以在组件中使用 mapMutations 辅助函数来将 mutation 映射到组件的方法中，从而方便组件调用 mutation。

<template>
  <div>
    <div>{{ count }}</div>
    <button @click="increment">Increment</button>
    <button @click="incrementBy(10)">Increment by 10</button>
  </div>
</template>

<script>
import { mapState, mapMutations } from 'vuex'

export default {
  computed: {
    ...mapState(['count'])
  },
  methods: {
    ...mapMutations(['increment', 'incrementBy'])
  }
}
</script>

在上面的代码中，我们使用 mapMutations 辅助函数将 increment 和 incrementBy mutation 映射到组件的方法中，从而可以在组件中直接调用这些 mutation。

需要注意的是，和 getter 一样，mutation 也可以接受其他 mutation 和组件中的参数，从而方便我们组合 mutation 和动态地修改 store 中的数据。

Mutation 是 Vuex 中用于同步修改 state 的核心概念。它可以看作是一个纯函数，接受 state 作为第一个参数，并接受一个可选的负载（payload）作为第二个参数，通过修改 state 的值来改变 store 中的数据。

下面我们来看一个实际的例子，假设我们有一个购物车应用，其中包含一个 cart 数组，用于存储用户选购的商品：

const store = new Vuex.Store({
  state: {
    cart: []
  },
  mutations: {
    addToCart(state, product) {
      state.cart.push(product)
    },
    removeFromCart(state, productId) {
      const index = state.cart.findIndex(item => item.id === productId)
      if (index !== -1) {
        state.cart.splice(index, 1)
      }
    },
    clearCart(state) {
      state.cart = []
    }
  }
})

在上面的代码中，我们定义了三个 mutation：addToCart、removeFromCart 和 clearCart。addToCart mutation 接受一个负载参数 product，其中包含要加入购物车的商品信息，它通过 state.cart.push(product) 将商品添加到 cart 数组中。removeFromCart mutation 接受一个负载参数 productId，其中包含要从购物车中删除的商品的 ID，它通过 state.cart.splice(index, 1) 将该商品从 cart 数组中删除。clearCart mutation 没有接受任何参数，它通过 state.cart = [] 将 cart 数组清空。

我们可以通过 store.commit 方法来触发 mutation：

store.commit('addToCart', { id: 1, name: 'product 1', price: 10 })
store.commit('removeFromCart', 1)
store.commit('clearCart')

在上面的代码中，我们分别通过 store.commit('addToCart', { id: 1, name: 'product 1', price: 10 })、store.commit('removeFromCart', 1) 和 store.commit('clearCart') 来触发 addToCart、removeFromCart 和 clearCart mutation。

除了直接调用 store.commit 方法，我们还可以在组件中使用 mapMutations 辅助函数来将 mutation 映射到组件的方法中，从而方便组件调用 mutation。

<template>
  <div>
    <div v-for="product in products" :key="product.id">
      {{ product.name }} - {{ product.price }}
      <button @click="addToCart(product)">Add to Cart</button>
      <button @click="removeFromCart(product.id)">Remove from Cart</button>
    </div>
    <button @click="clearCart">Clear Cart</button>
  </div>
</template>

<script>
import { mapState, mapMutations } from 'vuex'

export default {
  data() {
    return {
      products: [
        { id: 1, name: 'product 1', price: 10 },
        { id: 2, name: 'product 2', price: 20 },
        { id: 3, name: 'product 3', price: 30 }
      ]
    }
  },
  computed: {
    ...mapState(['cart'])
  },
  methods: {
    ...mapMutations(['addToCart', 'removeFromCart', 'clearCart'])
  }
  }
</script>

在上面的代码中，我们使用了 mapState 辅助函数将 cart 数组映射到组件的计算属性中，并使用了 mapMutations 辅助函数将 addToCart、removeFromCart 和 clearCart mutations 映射到组件的 methods 中。这样，我们就可以在组件中直接调用 this.addToCart(product)、this.removeFromCart(product.id) 和 this.clearCart() 来触发对应的 mutations 了。

需要注意的是，mutation 必须是同步函数，因为在 Vuex 中只有 mutation 才能修改 state，而且所有的 mutation 都是同步执行的。如果需要异步地修改 state，需要使用 action。

总结一下，Mutation 是 Vuex 中用于同步修改 state 的核心概念，它可以看作是一个纯函数，接受 state 作为第一个参数，并接受一个可选的负载作为第二个参数，通过修改 state 的值来改变 store 中的数据。在组件中可以通过直接调用 store.commit 方法或使用 mapMutations 辅助函数来触发 mutation。
