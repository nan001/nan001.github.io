# vuex4 测试

Vuex 4 提供了一些内置工具，例如 Vuex Test Utils，可用于编写单元测试和集成测试。

在编写 Vuex 4 测试时，我们通常需要测试以下几个方面：

State 是否正确获取或者修改 Getters 是否返回正确的值 Mutations 是否正确地改变 State 的值 Actions 是否正确地调用 Mutations 或者提交其他 Actions 下面是一个简单的 Vuex 4 测试示例：

import { createVuexStore, createStore } from 'vuex'
import { mount } from '@vue/test-utils'

// import the module to be tested
import myModule from '@/store/modules/myModule'

// create a store for testing
const store = createVuexStore({
  modules: {
    myModule
  }
})

describe('myModule', () => {
  it('returns the correct default state', () => {
    const defaultState = {
      foo: 'bar',
      count: 0
    }

    expect(store.state.myModule).toEqual(defaultState)
  })

  it('increments count using a mutation', () => {
    const count = store.state.myModule.count

    store.commit('myModule/incrementCount')

    expect(store.state.myModule.count).toBe(count + 1)
  })

  it('increments count using an action', async () => {
    const count = store.state.myModule.count

    await store.dispatch('myModule/incrementCountAsync')

    expect(store.state.myModule.count).toBe(count + 1)
  })
})

在这个示例中，我们首先导入了需要测试的 Vuex 模块，并创建了一个包含该模块的测试 store。接着，我们使用 Jest 中的 describe 和 it 函数来编写测试用例。

在第一个测试用例中，我们测试了 State 是否能正确获取到默认值。

在第二个测试用例中，我们测试了使用 Mutation 是否能正确修改 State 的值。

在第三个测试用例中，我们测试了使用 Action 是否能正确地调用 Mutation 并修改 State 的值。

这只是一个简单的测试示例，实际上我们可以在测试中编写更多的断言来测试不同的场景和组合。

在编写 Vuex 应用程序时，测试是非常重要的，因为测试可以帮助开发人员确保其代码在不同情况下都能够按预期工作。同时，测试还可以帮助开发人员迅速找到和修复潜在的问题。

在 Vuex 4 中，测试与之前版本的 Vuex 非常相似，主要使用 Jest 测试框架和 vue-test-utils 库来编写和运行测试。下面介绍一些常见的测试场景和用例。

# 测试 state 的初始值

测试 state 的初始值通常是我们测试 Vuex 应用程序的第一步。我们可以使用一个测试套件来测试所有 state 的初始值，或者为每个 state 编写一个单独的测试套件。

import { createStore } from 'vuex'
import myModule from './myModule'

describe('myModule', () => {
  test('state should have the correct initial values', () => {
    const store = createStore({
      modules: {
        myModule
      }
    })

    expect(store.state.myModule.myState).toBe(0)
  })
})

# 测试 mutation

测试 mutation 是 Vuex 测试中最常见的一种场景。我们可以使用 Jest 的 mock 方法来模拟 mutation 的行为，然后测试 mutation 的行为是否符合预期。

import { createStore } from 'vuex'
import myModule from './myModule'

describe('myModule', () => {
  test('INCREMENT_MY_STATE mutation should increment myState by 1', () => {
    const store = createStore({
      modules: {
        myModule
      }
    })

    // 模拟 INCREMENT_MY_STATE mutation
    store.commit('myModule/INCREMENT_MY_STATE')

    // 断言 myState 是否被正确地更新
    expect(store.state.myModule.myState).toBe(1)
  })
})

# 测试 action

测试 action 与测试 mutation 类似，也是使用 mock 方法来模拟 action 的行为，然后测试 action 的行为是否符合预期。同时，我们可以使用 async/await 关键字来测试异步 action。

import { createStore } from 'vuex'
import myModule from './myModule'

describe('myModule', () => {
  test('fetchData action should fetch data from API', async () => {
    const store = createStore({
      modules: {
        myModule
      }
    })

    // 模拟 fetchData action
    await store.dispatch('myModule/fetchData')

    // 断言 data 是否被正确地更新
    expect(store.state.myModule.data).toEqual({ id: 1, name: 'John' })
  })
})

# 测试 getter

测试 getter 也是比较常见的一种场景。我们可以使用 getters 属性来获取 getter 的值，并测试其行为是否符合预期。

import { createStore } from 'vuex'
import myModule from './myModule'

describe('myModule', () => {
  test('fullName getter should return the full name', () => {
    const store = createStore({
      modules: {
        myModule
      }
    })

    // 断言 fullName 是否返回正确的值
    expect(store.getter('myModule/fullName')({ firstName: 'John', lastName: 'Doe' })).toBe('John Doe')
})

test('setFirstName action should set the first name', () => {
const store = createStore({
modules: {
myModule
}
})

// dispatch setFirstName action
store.dispatch('myModule/setFirstName', 'Jane')

// 断言 state 中的 firstName 是否被正确设置
expect(store.state.myModule.firstName).toBe('Jane')
})
})

以上测试代码中，我们首先使用 createStore 创建了一个 Vuex Store，并在其中注册了 myModule 模块。然后，我们针对 myModule 模块中的 fullName getter 和 setFirstName action 分别进行了单元测试。

在第一个测试中，我们使用 store.getter 方法获取了 fullName getter，并使用 { firstName: 'John', lastName: 'Doe' } 作为参数调用它，最后使用 Jest 的 expect 断言判断返回值是否为 'John Doe'。

在第二个测试中，我们使用 store.dispatch 方法分发了 setFirstName action，并传入了 'Jane' 作为参数。最后，我们使用 Jest 的 expect 断言判断 store.state.myModule.firstName 是否被正确设置为 'Jane'。

在实际开发中，我们可以编写更多的测试用例来保证 Vuex Store 的正确性和稳定性。

当我们的 Vuex Store 变得越来越复杂时，编写和维护测试用例的难度也会越来越大。此时，我们可以考虑使用一些辅助工具来帮助我们更轻松地进行测试。

一个常用的辅助工具是 vuex-module-decorators 库，它提供了一些装饰器，可以帮助我们更方便地定义 Vuex 模块，并提供了一些实用的工具函数，例如 getModule 函数，可以帮助我们更方便地获取已注册的 Vuex 模块。

下面是一个使用 vuex-module-decorators 进行测试的例子：

import { createStore } from 'vuex'
import { Module, VuexModule, Mutation, Action } from 'vuex-module-decorators'
import { getModule } from 'vuex-module-decorators'

@Module({ namespaced: true, name: 'myModule', stateFactory: true })
class MyModule extends VuexModule {
  firstName = 'John'
  lastName = 'Doe'

  get fullName() {
    return `${this.firstName} ${this.lastName}`
  }

  @Mutation
  setFirstName(firstName: string) {
    this.firstName = firstName
  }

  @Action({ commit: 'setFirstName' })
  async updateFirstName(firstName: string) {
    // 模拟异步操作
    await new Promise(resolve => setTimeout(resolve, 1000))
    return firstName
  }
}

describe('MyModule', () => {
  test('fullName getter should return the full name', () => {
    const store = createStore({
      modules: {
        myModule: MyModule
      }
    })

    // 断言 fullName 是否返回正确的值
    expect(getModule(MyModule, store).fullName).toBe('John Doe')
  })

  test('updateFirstName action should update the first name', async () => {
    const store = createStore({
      modules: {
        myModule: MyModule
      }
    })

    // dispatch updateFirstName action
    const result = await getModule(MyModule, store).updateFirstName('Jane')

    // 断言 firstName 是否被正确设置
    expect(getModule(MyModule, store).firstName).toBe('Jane')

    // 断言 action 返回的值是否正确
    expect(result).toBe('Jane')
  })
})

在上述代码中，我们首先通过 @Module 装饰器定义了一个名为 MyModule 的 Vuex 模块，并在其中定义了 firstName、lastName、fullName、setFirstName 和 updateFirstName 等属性和方法。

接着，在测试用例中，我们使用 createStore 方法创建了一个 Vuex Store，并在其中注册了 MyModule 模块。然后，我们使用 getModule 函数获取了已注册的 MyModule 模块，并使用其提供的属性和方法进行单元测试。

在第一个测试中，我们使用 getModule(MyModule, store).fullName 获取了 fullName getter 的返回值，并使用 Jest 的 expect 断言判断返回值是否为 'John Doe'。

在第二个测试中，我们使用 getModule(MyModule, store).updateFirstName('Jane') 分发了 updateFirstName action，并对应的 mutation 是 updateFirstNameMutation。

测试 updateFirstName action 是否可以正确地更新 state 和 mutation 是否被正确地调用，可以这样写

test('updateFirstName action should update first name', () => {
  const store = createStore({
    modules: {
      myModule
    }
  })

  // 调用 action
  store.dispatch('myModule/updateFirstName', 'New First Name')

  // 断言 state 中的 firstName 是否被更新
  expect(store.state.myModule.firstName).toBe('New First Name')

  // 断言 mutation 是否被正确地调用
  expect(store.getters['myModule/fullName']).toBe('New First Name Last Name')
})

这个测试代码中，我们调用 updateFirstName action 并传递了 'New First Name' 参数，然后断言 state 中的 firstName 是否被更新成了 'New First Name'，接着通过调用 fullName getter 来断言 mutation 是否被正确地调用，fullName getter 会返回 firstName 和 lastName 拼接起来的结果，所以我们可以判断 fullName 是否等于 'New First Name Last Name' 来验证 mutation 是否被正确地调用。

这是一个简单的测试，更复杂的情况下可能需要模拟异步操作等等，但是这个测试代码可以帮助你理解如何测试 Vuex 模块。
