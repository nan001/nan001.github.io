# css3 媒体载体

# page

CSS3中的page模块是专门用于描述打印文档时的样式的，通过page模块，可以为不同的打印页面设置不同的样式，如页眉、页脚、页边距等。下面是page模块的详细介绍和示例：

CSS3中的page模块包括两个伪类：@page和@bottom-center、@bottom-left、@bottom-right、@top-center、@top-left、@top-right、@left-bottom、@left-middle、@left-top、@right-bottom、@right-middle、@right-top等12个伪类，用于定义打印页面的不同部位的样式。

@page伪类

@page伪类用于描述打印页面的整体样式，如页面大小、页边距、打印方向等，其语法格式如下：

@page {
  margin: 2cm;
  size: A4 landscape;
  /*其他样式*/
}

@bottom-center、@bottom-left、@bottom-right、@top-center、@top-left、@top-right、@left-bottom、@left-middle、@left-top、@right-bottom、@right-middle、@right-top伪类

这些伪类用于定义打印页面的不同部位的样式，如页眉、页脚等，其语法格式如下：

@page {
  @bottom-center {
    content: "页脚中心内容";
  }
  @top-left {
    content: "页眉左侧内容";
  }
  /*其他样式*/
}

下面是一个示例，展示如何使用@page和@bottom-center伪类设置打印页面的页脚：

@page {
  size: A4 portrait;
  margin: 2cm;
  @bottom-center {
    content: "页脚中心内容";
  }
}

在打印时，页面的底部中央将会显示“页脚中心内容”这个文本。

# width

CSS3中的width属性是用于定义元素的宽度的，它是一个常用的CSS属性之一。该属性可以设置元素的宽度值，有多种单位可供选择。

语法格式如下：

width: value;

其中，value可以是以下值之一：

length：具体的长度值，如100px。
percentage：相对于包含块的百分比值，如50%。
auto：浏览器会计算出元素的宽度。
inherit：继承父元素的宽度值。

示例：

<!DOCTYPE html>
<html>
<head>
	<title>width property example</title>
	<style>
		.box1 {
			width: 200px;
			height: 100px;
			background-color: red;
		}
		.box2 {
			width: 50%;
			height: 100px;
			background-color: green;
		}
	</style>
</head>
<body>
	<div class="box1"></div>
	<div class="box2"></div>
</body>
</html>

在上面的示例中，我们定义了两个div元素，它们分别有不同的宽度值。其中，box1的宽度为200px，box2的宽度为50%。

# height

height 属性用于设置元素的高度。

语法

height: auto|length|initial|inherit;

属性值：

auto：元素的高度由其内容撑开（默认值）。
length：具体的高度值，比如 px、em、rem 等。
initial：将属性值设置为默认值。
inherit：继承父元素的属性值。

示例 设置元素的高度为 100px：

div {
  height: 100px;
}

注意事项 如果使用百分比作为属性值，元素的高度将基于其父元素的高度。但是如果其父元素的高度为 auto，则百分比无法生效。

height 属性也可以用于替换元素，如 img 和 object 等。

当设置元素的高度小于其内容的高度时，内容会溢出元素。

height 属性仅适用于块级元素和替换元素。如果应用于行内元素，则会被忽略。

# device-width

在CSS中，device-width是一个非常常用的视口单位，它指定了设备屏幕的宽度。设备宽度可以用CSS中的@media查询来检测，以便在不同的设备上应用不同的CSS样式。device-width单位通常在响应式设计中使用，以确保网站在不同设备上都具有良好的可用性。

以下是一个使用device-width的示例：

<head>
  <style>
    /* 应用于宽度小于等于 500px 的设备 */
    @media (max-width: 500px) {
      body {
        font-size: 16px;
      }
    }
    /* 应用于宽度大于 500px 的设备 */
    @media (min-width: 501px) {
      body {
        font-size: 24px;
      }
    }
  </style>
</head>
<body>
  <p>Hello, world!</p>
</body>

在这个示例中，我们使用@media查询和device-width来设置不同屏幕宽度下的不同字体大小。如果设备的宽度小于等于500px，字体大小将设置为16px，否则将设置为24px。

请注意，device-width并不是一个CSS属性，而是一个用于描述设备宽度的视口单位。因此，它只能在CSS的@media查询中使用。

# device-height

device-height是CSS3中一个用于媒体查询的属性，用于匹配设备高度的大小。它可以指定视口的高度，从而使得Web开发者可以通过媒体查询来适应各种不同高度的设备。

下面是一个示例：

<!DOCTYPE html>
<html>
<head>
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <style>
        body {
            background-color: white;
        }

        @media screen and (device-height: 736px) {
            body {
                background-color: orange;
            }
        }

        @media screen and (device-height: 667px) {
            body {
                background-color: yellow;
            }
        }

        @media screen and (device-height: 568px) {
            body {
                background-color: green;
            }
        }

        @media screen and (device-height: 480px) {
            body {
                background-color: blue;
            }
        }
    </style>
</head>
<body>
    <h1>Device Height Test</h1>
</body>
</html>

在上面的示例中，我们使用@media媒体查询来根据设备的高度来设置不同的背景颜色。这样，当页面加载时，它将根据设备的高度来自动调整背景颜色。

# orientation

CSS3中的orientation属性用于根据设备的方向设置样式。它可以检测设备的方向，并根据情况改变页面的布局和外观。

orientation属性接受两个值：

portrait：表示设备处于纵向模式（默认值）。
landscape：表示设备处于横向模式。

下面是一个示例，展示如何在横向模式下设置不同的样式：

<!DOCTYPE html>
<html>
  <head>
    <style>
      /* 在横向模式下设置不同的背景颜色 */
      @media screen and (orientation: landscape) {
        body {
          background-color: lightblue;
        }
      }
    </style>
  </head>
  <body>
    <h1>Hello, World!</h1>
  </body>
</html>

当设备处于纵向模式时，页面的背景颜色将是默认颜色，而当设备处于横向模式时，页面的背景颜色将变为lightblue。

# aspect-ratio

aspect-ratio是CSS3新增的一个用于设置宽高比的属性，它可以让我们在保持宽高比不变的情况下，灵活地控制元素的宽度或高度。具体来说，aspect-ratio属性可以接受两个值，第一个值表示宽度，第二个值表示高度，例如：

div {
  aspect-ratio: 16 / 9;
}

这个例子中，我们将一个div元素的宽高比设置为16:9，也就是说，宽度是高度的1.777倍。当我们改变div元素的宽度或高度时，它的高度或宽度将根据宽高比自动调整。

需要注意的是，aspect-ratio属性目前处于实验阶段，不是所有的浏览器都支持。在使用时，需要添加浏览器前缀，例如：

div {
  -webkit-aspect-ratio: 16 / 9;
  aspect-ratio: 16 / 9;
}

另外，aspect-ratio属性只能用于块级元素，不能用于内联元素。

# device-aspect-ratio

device-aspect-ratio是CSS3媒体查询中用于描述设备宽高比的属性。它用于匹配特定的设备宽高比，允许您针对不同的设备和屏幕尺寸应用不同的样式。

device-aspect-ratio属性接受两个值，表示允许的最小和最大设备宽高比，可以是浮点数、整数或百分比。

下面是一些示例：

/* 匹配设备宽高比为16:9的设备 */
@media screen and (device-aspect-ratio: 16/9) {
  /* 应用样式 */
}

/* 匹配设备宽高比大于1.5的设备 */
@media screen and (min-device-aspect-ratio: 1.5) {
  /* 应用样式 */
}

/* 匹配设备宽高比小于2的设备 */
@media screen and (max-device-aspect-ratio: 2) {
  /* 应用样式 */
}

值得注意的是，device-aspect-ratio只会匹配设备的物理宽高比，而不是浏览器窗口的宽高比。因此，在使用时要根据实际情况选择使用。

# color

CSS3中的color属性用于设置文本颜色。在CSS3中，可以使用以下方式来表示颜色：

通过颜色名称或关键字（如red，green等）来指定颜色
通过RGB（红绿蓝）值来指定颜色
通过十六进制值来指定颜色
通过HSL（色相，饱和度，亮度）值来指定颜色

下面是一些color属性的示例：

指定红色文本：

color: red;

指定RGB值为(255, 0, 0)的红色文本：

color: rgb(255, 0, 0);

指定十六进制值为#FF0000的红色文本：

color: #FF0000;

指定HSL值为(0, 100%, 50%)的红色文本：

color: hsl(0, 100%, 50%);

# color-index

color-index 是 CSS3 中的一个媒体查询，用于选择设备支持的色彩数量。

color-index 的取值为一个正整数，代表设备支持的颜色数目。如果设备支持的颜色数目小于指定值，则该样式不会被应用。

以下是一个示例：

@media screen and (color-index: 256) {
  body {
    background-color: #ffffff;
  }
}

上述代码表示，如果设备支持 256 种颜色，则页面背景色为白色（#ffffff）。

在实际应用中，使用 color-index 可以根据设备的颜色支持情况，针对不同的设备提供不同的颜色方案，以保证页面的可读性和可访问性。

# monochrome

CSS3的monochrome媒体查询是用于匹配单色（黑白）显示设备的查询。它可以用来检查设备是否仅支持黑色和白色的颜色，而不支持其他颜色。

monochrome媒体查询接受一个min或max关键字，后跟单色的位数，以及可选的分辨率条件。以下是一个示例：

@media (min-monochrome: 2) {
  /* styles applied for devices with at least 2 bits of monochrome */
}

@media (max-monochrome: 1) {
  /* styles applied for devices with at most 1 bit of monochrome */
}

@media (min-monochrome: 2) and (max-resolution: 300dpi) {
  /* styles applied for devices with at least 2 bits of monochrome and a maximum resolution of 300dpi */
}

上面的代码演示了如何根据单色的位数和分辨率条件应用不同的CSS样式。例如，第一个媒体查询将仅应用于具有至少2位单色的设备，而第二个媒体查询将仅应用于仅支持单色的设备。

# resolution

CSS3中的resolution属性用于指定输出设备的分辨率，单位为dpi（每英寸点数）。它通常用于打印样式表，以确保生成的文档具有所需的打印质量。

语法：

@media print and (resolution: 300dpi) {
  /* 打印时应用的样式 */
}

示例：

@media print and (min-resolution: 300dpi) and (max-resolution: 600dpi) {
  /* 当分辨率在300dpi和600dpi之间时应用的样式 */
}

# scan

CSS3中的scan媒体查询用于检测媒体的扫描方式，也就是电视或监视器的刷新方式。它有以下两个值可用：

interlace：指定扫描方式为隔行扫描。 progressive：指定扫描方式为逐行扫描。 以下是一个例子：

@media screen and (scan: progressive) {
  body {
    background-color: #f2f2f2;
  }
}

上述代码将检测媒体扫描方式是否为逐行扫描，如果是则将文档背景颜色设置为#f2f2f2。

# grid

CSS3中的网格布局（Grid Layout）是一个强大的布局系统，它可以用于创建复杂的、多列的、多行的网格布局。网格布局是一个二维布局系统，它允许您在网格中放置元素，而不需要使用复杂的浮动、定位和清除技巧。在网格布局中，您可以指定元素的行和列位置，也可以指定它们的大小和对齐方式。

# grid属性

网格布局由以下属性组成：

display：指定元素应该采用网格布局，可以设置为 grid、inline-grid、subgrid。
grid-template-columns 和 grid-template-rows：定义网格的列和行。
grid-template-areas：定义网格的区域。
grid-template：简写属性，包括 grid-template-rows、grid-template-columns 和 grid-template-areas。
grid-column-gap 和 grid-row-gap：定义网格行和列之间的间距。
grid-gap：简写属性，包括 grid-row-gap 和 grid-column-gap。
justify-items 和 align-items：定义网格项目在网格单元格中的对齐方式。
justify-content 和 align-content：定义网格的对齐方式。
grid-auto-rows 和 grid-auto-columns：定义自动行和列的大小。
grid-auto-flow：定义项目如何放置在自动网格轨道上。

# grid-template-columns 和 grid-template-rows

grid-template-columns 和 grid-template-rows 属性用于定义网格的列和行。您可以为每个列或行指定一个长度或一个百分比，也可以使用关键字来定义网格的大小。

例如，以下 CSS 代码定义了一个 3 列的网格，每列的宽度分别为 200 像素、1 个自适应列和 2 个自适应列。

.container {
  display: grid;
  grid-template-columns: 200px 1fr 2fr;
}

如果您想要创建一个有 5 行、每行高度为 100 像素的网格，可以使用以下 CSS 代码：

.container {
  display: grid;
  grid-template-rows: repeat(5, 100px);
}

# grid-template-areas

grid-template-areas 属性用于定义网格的区域。您可以为每个单元格指定一个名称，然后在网格容器中将这些名称组合成一个网格。

例如，以下 CSS 代码定义了一个 3x3 的网格，每个单元格都有一个名称：

.container {
  display: grid;
  grid-template-areas:
    "header header header"
    "main main sidebar"
    "footer footer footer";
}

然后，您可以使用 grid-area 属性为每个网格项目指定其所属的区域：

.item-1 {
  grid-area: header;
}

.item-2 {
  grid-area: main;
}

.item-3 {
  grid-area: sidebar;
}

.item-4 {
  grid-area: footer;
}

# grid-template

grid-template 属性是一个简写属性，包括grid-template-columns、grid-template-rows 和 grid-template-areas 这三个属性。

grid-template-columns：定义网格列的数量和宽度。
grid-template-rows：定义网格行的数量和高度。
grid-template-areas：指定网格区域的名称。

示例：

.grid-container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  grid-template-rows: 50px 100px 50px;
  grid-template-areas: 
    "header header header"
    "sidebar content content"
    "sidebar footer footer";
}

这个示例定义了一个具有三个列和三个行的网格容器。第一行有一个高度为 50 像素的单元格，第二行有一个高度为 100 像素的单元格，第三行有一个高度为 50 像素的单元格。第一列和第三列的宽度都是 1/3，第二列的宽度是剩余空间的 1/3。通过 grid-template-areas 属性，我们指定了每个单元格的名称，这些名称可以在 grid-area 属性中使用，以指定每个单元格的位置。

# -webkit-text-fill-color

-webkit-text-fill-color是一个Webkit引擎私有的CSS属性，用于设置文本填充颜色。它的作用类似于color属性，但具有更高的优先级，可以覆盖某些情况下的color属性。

语法：

-webkit-text-fill-color: <color>

其中，<color>可以是CSS中支持的任何一种颜色值，如关键字、RGB、十六进制值等。

示例：

h1 {
  -webkit-text-fill-color: red;
}

在上面的例子中，h1元素的文本填充颜色将被设置为红色。

需要注意的是，由于-webkit-text-fill-color是一个非标准的私有属性，因此不应该在生产环境中过度依赖它。在某些情况下，某些浏览器可能不支持它，或者可能在未来的版本中被删除或更改。

# -webkit-text-stroke

-webkit-text-stroke是一个Webkit私有属性，它可以用来为文本添加描边效果，即在文本周围添加一定宽度和颜色的描边。

语法如下：

-webkit-text-stroke: width color;

其中：

width指定描边的宽度，可以是正数、负数或零。正数表示描边在文本周围，负数表示描边在文本内部，零表示没有描边。默认值为0。
color指定描边的颜色。可以使用任何CSS颜色值。

注意事项：

由于这是一个Webkit私有属性，因此在其他浏览器中不被支持。
当-webkit-text-fill-color和-webkit-text-stroke同时存在时，-webkit-text-fill-color优先于-webkit-text-stroke。

# -webkit-text-stroke-width

-webkit-text-stroke-width 是一个 WebKit 特有的 CSS 属性，用于为文本描边设置宽度。它可以接受一个长度值作为描边宽度，单位可以是 px、em 等。

示例：

h1 {
  font-size: 4rem;
  font-weight: bold;
  color: white;
  -webkit-text-stroke: 2px black;
  -webkit-text-fill-color: white;
}

上面的代码将 h1 元素中的文本进行描边，并将描边宽度设置为 2 像素，颜色设置为黑色。同时，将文本的颜色设置为白色。

需要注意的是，该属性是 WebKit 特有的，只在 Safari 和 Chrome 中支持。在其他浏览器中，可以使用 text-stroke 属性来实现类似的效果，但是语法略有不同。

# -webkit-text-stroke-color

-webkit-text-stroke-color是一个CSS属性，用于设置文本描边的颜色。该属性只在WebKit浏览器（例如Safari和Chrome）中支持，并且需要与-webkit-text-stroke-width属性一起使用。

以下是该属性的语法：

-webkit-text-stroke-color: <color> | inherit;

其中，<color>表示要设置的描边颜色，可以是CSS颜色值、RGB值、十六进制值等。inherit表示从父元素继承该属性。

以下是一个示例：

<p class="text">Hello World</p>

.text {
  font-size: 48px;
  font-weight: bold;
  color: #fff;
  -webkit-text-stroke-width: 2px;
  -webkit-text-stroke-color: #000;
}

在上面的示例中，文本的颜色为白色，描边的宽度为2像素，描边的颜色为黑色。这将创建一个有轮廓的白色文本效果。

# -webkit-box-reflect

-webkit-box-reflect 是一个非标准的 CSS 属性，只能在 WebKit 内核的浏览器中使用，用于创建一个指定元素的镜像效果。

语法：

-webkit-box-reflect: <direction> <offset> <mask-box-image>;

其中：

direction：必选参数，表示镜像的方向，可以取值 above（上）、below（下）、left（左）和 right（右）。
offset：必选参数，表示镜像的偏移距离。
mask-box-image：可选参数，表示镜像的遮罩图片，可以使用 url() 指定图片的地址。

示例：

.reflect {
  -webkit-box-reflect: below 1px linear-gradient(to bottom, rgba(255,255,255,0.2) 0%, rgba(255,255,255,0) 100%);
}

该示例为指定的元素 .reflect 添加了一个下方的镜像效果。其中，below 表示镜像方向为下方，1px 表示镜像距离元素底部的偏移距离，linear-gradient() 表示使用线性渐变来遮罩镜像，从而实现类似于半透明的镜像效果。

注意：该属性只能在 WebKit 内核的浏览器中使用，其他浏览器不支持，且可能在未来会被废弃。建议谨慎使用。

# -webkit-tap-highlight-color

-webkit-tap-highlight-color是一个用于移动设备的Webkit私有属性，用于设置当元素被点击时，元素周围的高亮颜色。在移动设备上，点击一个元素时，通常会出现一个蓝色的高亮色，这个属性可以改变这个高亮颜色。

该属性只在Webkit浏览器中有效，并且仅适用于移动设备。在桌面浏览器中设置该属性不会有任何效果。

语法：

-webkit-tap-highlight-color: <color>;

该属性只有一个属性值：<color>，用于指定高亮的颜色。

示例：

/* 将高亮颜色设置为红色 */
-webkit-tap-highlight-color: red;

# -webkit-user-drag

-webkit-user-drag是一个CSS属性，用于指定元素是否可拖动。它只适用于Webkit浏览器，如Safari和Chrome。

语法：

-webkit-user-drag: none | element | auto;

取值：

none：禁止元素拖动；
element：元素内的内容可以拖动；
auto：默认值，元素和内容都可以拖动。

示例：

<div style="-webkit-user-drag: element;">
  <p>这是一个可拖动的段落。</p>
</div>

注意：该属性只能在Webkit浏览器中使用，因此如果需要兼容多个浏览器，最好使用更通用的属性，如draggable属性。

# -webkit-overflow-scrolling

-webkit-overflow-scrolling 是一个用于移动设备上的 CSS 属性，可以用来改善页面滚动时的体验。它仅适用于 WebKit 内核的浏览器，比如 Safari 和 Chrome。

该属性主要用于控制容器元素的滚动方式，可以设置为 auto、touch 或 scroll。

auto：使用普通的滚动方式，即滚动停止后立即停止滚动。
scroll：在滚动时使用硬件加速，滚动停止后会继续滚动一段距离再停止。
touch：在滚动时使用软件加速，滚动停止后会继续滚动一段距离再停止。此模式下的滚动体验最流畅，但是会消耗更多的 CPU 资源。

例如，下面的代码将 .scroll-container 元素的滚动方式设置为软件加速模式：

.scroll-container {
  -webkit-overflow-scrolling: touch;
}

需要注意的是，-webkit-overflow-scrolling 只对 overflow: scroll 或 overflow: auto 的容器元素生效。此外，如果容器元素设置了 -webkit-transform，则该属性可能会失效。

# -webkit-line-clamp

-webkit-line-clamp 是一个 WebKit 私有的 CSS 属性，可以用于限制文本块在指定行数内显示，超出部分被省略号（...）替代。

语法：

-webkit-line-clamp: <number>

属性值：

<number>：指定文本块应该被显示的行数。必须是一个非负整数。 示例：

<style>
  .text {
    display: -webkit-box;
    -webkit-line-clamp: 3;
    -webkit-box-orient: vertical;
    overflow: hidden;
    text-overflow: ellipsis;
  }
</style>
<div class="text">
  这是一个文本块，可以通过设置 -webkit-line-clamp 属性来控制在指定的行数内显示，并在超出的部分显示省略号。
</div>

解释：

上面的示例中，我们定义了一个文本块 .text，使用了 -webkit-line-clamp 属性，将文本块限制在 3 行以内显示。如果文本块超过了 3 行，将会被省略号替代。同时，我们也设置了 overflow 和 text-overflow 属性，用于控制文本溢出时的处理方式。注意，-webkit-line-clamp 是 WebKit 私有的属性，只在 Safari 和 Chrome 等使用 WebKit 引擎的浏览器中生效。在其他浏览器中可能需要使用 -webkit-box-orient 和 -webkit-box-reflect 等属性来实现类似的效果。
