# webpack 初体验

# 安装

# 1、安装Node.js和npm。如果您已经安装了Node.js和npm，可以跳过此步骤。

# 2、使用npm全局安装Webpack 4和Webpack CLI：

npm install -g webpack webpack-cli

# 3、创建一个新的项目文件夹并进入该文件夹：

mkdir my-webpack-project
cd my-webpack-project

# 4、在项目文件夹中初始化npm：

npm init -y

# 5、安装需要的依赖（例如，用于处理JavaScript和CSS的loader）：

npm install --save-dev webpack webpack-cli webpack-dev-server babel-loader css-loader style-loader

# 使用

# 1、创建一个src文件夹，并在其中创建一个index.js和一个style.css文件：

index.js：

console.log('Hello, webpack!');

style.css：

body {
  background-color: blue;
}

# 2、创建一个webpack.config.js文件，并配置Webpack：

const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist')
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: ['@babel/preset-env']
          }
        }
      },
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader']
      }
    ]
  },
  devServer: {
    contentBase: './dist'
  }
};

这个配置文件告诉Webpack将src/index.js作为入口文件，并将输出文件保存为dist/bundle.js。它还告诉Webpack使用babel-loader来处理JavaScript，并使用style-loader和css-loader来处理CSS。devServer配置项指定Web服务器从哪个目录提供文件。

# 3、在package.json中添加脚本命令：

"scripts": {
  "start": "webpack serve --mode development",
  "build": "webpack --mode production"
}

这些脚本命令将启动Webpack开发服务器（用于本地开发）或生成一个生产版本的应用程序。

# 4、在命令行中执行npm start命令

然后打开浏览器并访问http://localhost:8080。您应该可以看到“Hello, webpack!”的输出，并且背景颜色应为蓝色。

# 5、在命令行中执行npm run build命令

然后检查dist文件夹中是否生成了bundle.js文件。这个文件是Webpack打包后的输出文件，可以用于生产环境部署。
