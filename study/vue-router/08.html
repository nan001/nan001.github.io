# vue-router4 自动激活 CSS 类的链接

在 Vue Router 4 中，我们可以使用 linkExactActiveClass 和 linkActiveClass 两个属性来自动激活 CSS 类的链接。

linkExactActiveClass 用于设置精确匹配路由时应该添加的 CSS 类，默认值是 "router-link-exact-active"。

linkActiveClass 用于设置模糊匹配路由时应该添加的 CSS 类，默认值是 "router-link-active"。

我们可以在创建 Router 实例时将这两个属性传递给路由配置对象，例如：

const router = createRouter({
  history: createWebHistory(),
  routes: [
    {
      path: '/',
      name: 'home',
      component: Home,
      meta: { title: 'Home' },
      // 自动激活 CSS 类的链接
      linkExactActiveClass: 'active',
      linkActiveClass: 'active'
    },
    // ...
  ]
})

在上面的代码中，我们将 linkExactActiveClass 和 linkActiveClass 都设置为 "active"，表示当链接精确匹配或模糊匹配当前路由时，会自动添加 active 类到链接上。

当然，我们也可以为不同的路由配置设置不同的 CSS 类，只需要在对应的路由配置中指定即可。

这两个属性可以在路由实例创建时全局配置，也可以在单个路由配置中进行覆盖。例如：

const router = createRouter({
  routes,
  linkActiveClass: 'active',
  linkExactActiveClass: 'exact-active',
});

<!-- 使用全局配置的 class -->
<router-link to="/" class="nav-link">Home</router-link>
<router-link to="/about" class="nav-link">About</router-link>

<!-- 覆盖全局配置的 class -->
<router-link to="/" class="nav-link" active-class="selected">Home</router-link>
<router-link to="/about" class="nav-link" exact-active-class="exact-selected">About</router-link>

还有一种方式是使用 router-link-exact-active 类名，它表示当前路径和激活的路由路径完全相同。例如，下面的代码会将“首页”这个链接在访问根路径时自动激活：

<router-link to="/" exact>首页</router-link>

这个链接只有在访问根路径时才会激活，因为只有根路径和路径 / 完全相同。

此外，你也可以通过 exact-active-class 属性来自定义激活时的类名。例如：

<router-link to="/" exact active-class="my-active-class">首页</router-link>

这里的 my-active-class 就是自定义的激活时的类名。

总之，<router-link> 提供了多种方式来自动激活 CSS 类的链接，可以根据实际情况选择合适的方式。
