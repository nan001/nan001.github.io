# vue-router4 可定制的滚动行为

Vue Router 4 可以通过 scrollBehavior 选项来定制路由切换时页面滚动的行为。默认情况下，路由切换时会让页面滚动到顶部，但是在一些场景中，我们可能需要定制滚动行为以实现更好的用户体验。

scrollBehavior 选项接收一个函数，该函数接收三个参数：前一个路由对象、当前路由对象、以及保存滚动位置的对象。函数需要返回一个包含滚动位置信息的对象，该对象可以包含以下属性：

x：水平方向滚动的位置 y：垂直方向滚动的位置 selector：选择器字符串，用于选择需要滚动到的元素 以下是一个简单的例子，其中通过判断当前路由对象是否带有 hash 属性来确定是否滚动到锚点位置：

const router = createRouter({
  history: createWebHistory(),
  scrollBehavior(to, from, savedPosition) {
    if (to.hash) {
      return { selector: to.hash };
    } else {
      return { top: 0 };
    }
  },
  routes: [
    // 路由配置
  ],
});

在上面的例子中，当当前路由对象带有 hash 属性时，返回一个包含 selector 属性的对象，用于选择需要滚动到的元素；否则返回一个包含 top 属性的对象，用于滚动到页面顶部。

除了上述示例，我们还可以根据具体的需求来定制滚动行为，例如：保存滚动位置，实现页面的历史滚动记录，滚动到指定元素等。

Vue Router 4 可定制的滚动行为还支持通过函数动态设置滚动行为。这意味着你可以根据路由对象的不同属性来定制每个路由的滚动行为，例如，根据路由的 params 来设置滚动位置。以下是一个使用函数定制滚动行为的示例：

const router = createRouter({
  history: createWebHistory(),
  scrollBehavior(to, from, savedPosition) {
    // 判断是否有 savedPosition，有的话就直接返回保存的值
    if (savedPosition) {
      return savedPosition
    } else {
      // 定义一个空对象
      let position = {}
      // 判断路由中是否设置了 meta 中的 scrollToTop
      if (to.matched.some(m => m.meta.scrollToTop)) {
        // 设置滚动位置为顶部
        position = { top: 0 }
      }
      // 如果路由中包含 hash，就把 hash 作为滚动位置
      if (to.hash) {
        position = { selector: to.hash }
      }
      // 返回滚动位置
      return position
    }
  },
  routes: [
    // 路由配置
  ]
})

在上面的示例中，我们通过 scrollBehavior 函数来设置滚动行为。首先，我们判断是否有保存的滚动位置，如果有，就直接返回保存的值；如果没有，则根据不同的条件来定制滚动位置。在这个例子中，我们判断路由中是否设置了 meta 中的 scrollToTop 属性，如果设置了，则将滚动位置设置为顶部；如果路由中包含 hash，则将 hash 作为滚动位置。最后，我们将滚动位置作为对象返回。

# 保存滚动位置

Vue Router 4 允许我们在路由之间保持滚动位置，这是一个非常实用的功能，可以增强用户体验。当用户在一个页面滚动一段距离，然后切换到另一个页面再回来时，页面应该自动滚动到之前的位置。下面我们就来详细介绍如何在 Vue Router 4 中实现保存滚动位置的功能。

首先，我们需要在路由配置中添加 scrollBehavior 函数来处理滚动行为。这个函数接收三个参数：

to：要进入的目标路由对象。 from：离开的路由对象。 savedPosition：通过 history.back() 或 history.forward() 之后，页面应该滚动到的位置。 scrollBehavior 函数需要返回一个对象，用于指定滚动的行为。我们可以使用 x 和 y 属性来指定滚动的位置。例如，要将页面滚动到顶部，我们可以返回 { x: 0, y: 0 }。

下面是一个基本的 scrollBehavior 函数：

const router = createRouter({
  history: createWebHistory(),
  scrollBehavior(to, from, savedPosition) {
    if (savedPosition) {
      return savedPosition
    } else {
      return { x: 0, y: 0 }
    }
  },
  routes: [
    // ...
  ]
})

这个函数检查 savedPosition 参数，如果存在，则返回该值以恢复之前的滚动位置。否则，它将返回 { x: 0, y: 0 }，将页面滚动到顶部。

但是，这个函数只能处理简单的滚动情况。如果要处理更复杂的情况，例如滚动到特定元素或处理动画效果等，则需要进行一些额外的工作。

一种方法是在路由配置中为每个路由指定一个 scrollTo 属性，用于指定要滚动到的元素的选择器或滚动位置。然后在 scrollBehavior 函数中根据 to 对象查找该值，并根据需要滚动到该元素或位置。

例如，要将页面滚动到具有 id 为 about 的元素上，我们可以将 scrollTo 属性添加到 about 路由中：

const router = createRouter({
  history: createWebHistory(),
  scrollBehavior(to, from, savedPosition) {
    if (to.hash) {
      return { selector: to.hash }
    } else if (to.matched.some(record => record.meta.scrollTo)) {
      return { selector: to.meta.scrollTo }
    } else if (savedPosition) {
      return savedPosition
    } else {
      return { x: 0, y: 0 }
    }
  },
  routes: [
    {
      path: '/about',
      name: 'About',
      component: About,
      meta: {
        scrollTo: '#about'
      }
    },
    // ...
  ]
})

然后在 scrollBehavior 函数中查找 meta.scrollTo，如果找到则执行 scrollToTarget 函数，传入元素的选择器作为参数。示例代码如下：

const router = createRouter({
  history: createWebHistory(),
  routes,
  scrollBehavior(to, from, savedPosition) {
    if (savedPosition) {
      return savedPosition
    } else {
      const position = {}
      if (to.hash) {
        position.el = to.hash
      }
      if (to.matched.some((m) => m.meta.scrollTo)) {
        const selector = to.matched.find((m) => m.meta.scrollTo).meta.scrollTo
        return scrollToTarget(selector, position)
      } else {
        return position
      }
    }
  },
})

最后，在需要滚动到指定元素的路由中，我们可以在 meta 中添加 scrollTo 字段，将需要滚动到的元素的选择器作为值。例如，我们要在 /blog/:id 页面滚动到 id 为 comment 的元素，可以这样配置：

{
  path: '/blog/:id',
  name: 'Blog',
  component: Blog,
  meta: {
    scrollTo: '#comment'
  }
}

这样，每次访问 /blog/:id 页面时，Vue Router 将自动滚动到 id 为 comment 的元素位置。

# 实现页面的历史滚动记录

Vue Router 4 提供了一个内置函数 scrollBehavior，用于定义路由切换时滚动行为。通过这个函数，我们可以自定义路由切换时的滚动行为，并且可以实现页面的历史滚动记录。

默认情况下，当路由切换时，页面会滚动到顶部。但是，在某些情况下，我们希望页面在路由切换后能够自动滚动到之前滚动的位置，或者在路由切换后滚动到某个特定的元素位置。

为了实现这个功能，我们可以在 scrollBehavior 函数中保存每个页面的滚动位置，并在路由切换时恢复滚动位置。

下面是一个示例代码，演示了如何在 scrollBehavior 函数中保存和恢复滚动位置：

const router = new VueRouter({
  mode: 'history',
  scrollBehavior(to, from, savedPosition) {
    // 如果当前页面存在滚动位置，则恢复滚动位置
    if (savedPosition) {
      return savedPosition
    }
    // 查找要滚动到的元素
    const selector = to.hash || to.meta.scrollTo
    const target = document.querySelector(selector)
    if (target) {
      return { selector, offset: { x: 0, y: 50 } }
    }
    // 默认滚动到页面顶部
    return { x: 0, y: 0 }
  }
})

在这个示例中，我们首先检查 savedPosition 参数是否存在。如果存在，说明我们可以恢复之前保存的滚动位置，直接返回即可。

如果 savedPosition 不存在，则查找要滚动到的元素。如果目标元素存在，则返回包含该元素选择器和偏移量的位置对象。否则，返回一个默认的滚动位置对象，使页面滚动到顶部。

当然，为了能够在路由切换时保存滚动位置，我们需要在每个页面组件的 beforeRouteLeave 钩子中保存滚动位置，如下所示：

export default {
  name: 'MyComponent',
  beforeRouteLeave(to, from, next) {
    // 保存滚动位置
    window.sessionStorage.setItem('scrollPosition', JSON.stringify(window.pageYOffset))
    next()
  }
}

在该钩子中，我们可以将当前页面的滚动位置保存到本地存储中，以便在下一次加载该页面时可以恢复滚动位置。

最后，我们还需要在页面组件的 mounted 钩子中恢复滚动位置，如下所示：

export default {
  name: 'MyComponent',
  mounted() {
    // 恢复滚动位置
    const scrollPosition = JSON.parse(window.sessionStorage.getItem('scrollPosition'))
    if (scrollPosition !== null) {
      window.scrollTo(0, scrollPosition)
      window.sessionStorage.removeItem('scrollPosition')
    }
  }
}

在该钩子中，我们首先从本地存储中获取已经保存的滚动位置信息，如果有，则将滚动条滚动到该位置，否则不进行任何操作。同时，我们需要在每个路由切换完成后将当前的滚动位置信息保存到本地存储中，以便后续的页面访问时能够恢复到上次的滚动位置。我们可以使用 window.localStorage 来进行本地存储的操作，例如：

const router = new VueRouter({
  routes,
  scrollBehavior(to, from, savedPosition) {
    // 如果 savedPosition 存在，直接滚动到相应位置
    if (savedPosition) {
      return savedPosition;
    } else {
      // 否则检查路由是否有 meta.scrollTo 属性
      const scrollTo = to.meta.scrollTo;
      if (scrollTo) {
        // 如果有，则滚动到指定选择器所在的元素
        return { selector: scrollTo };
      } else {
        // 否则不进行任何滚动
        return { x: 0, y: 0 };
      }
    }
  }
});

// 在路由切换完成后保存滚动位置信息到本地存储中
router.afterEach((to, from) => {
  const scrollPosition = { x: window.pageXOffset, y: window.pageYOffset };
  window.localStorage.setItem(to.fullPath, JSON.stringify(scrollPosition));
});

在上述代码中，我们使用了 window.localStorage.setItem() 方法来将当前的滚动位置信息保存到本地存储中，并使用 JSON.stringify() 方法将其转换为 JSON 字符串。同时，我们还在 scrollBehavior 函数中使用了 window.localStorage.getItem() 方法来从本地存储中获取已经保存的滚动位置信息，并使用 JSON.parse() 方法将其转换为 JavaScript 对象。最后，我们将该对象作为滚动位置信息返回，以便页面滚动到相应位置。

# 滚动到指定元素

Vue Router 4 也提供了一种方法来滚动到指定的元素。这可以通过在路由导航时设置一个选择器来实现。具体实现步骤如下：

# 在路由配置中设置 scrollBehavior 函数，以便在导航时保存滚动位置，如下所示：

const router = createRouter({
  history: createWebHistory(),
  routes: [...],
  scrollBehavior(to, from, savedPosition) {
    if (to.hash) {
      return { selector: to.hash }
    } else if (savedPosition) {
      return savedPosition
    } else {
      return { x: 0, y: 0 }
    }
  }
})

# 在需要滚动到的元素上设置一个 ID，例如：

<div id="target-element">This is the element to scroll to.</div>

# 导航到该元素所在的路由，并在 URL 中添加元素 ID，例如：

router.push({ name: 'my-route', hash: '#target-element' })

这将滚动到 ID 为 "target-element" 的元素，并且 URL 将显示 "#target-element"。

注意：滚动到元素只在页面加载时发生。如果需要在同一页面内滚动到指定元素，请使用 Vue.js 的 $refs 来获取该元素并使用 JavaScript 滚动到该元素。
