# vue-router4 嵌套路由映射

Vue Router 4 中，嵌套路由映射指的是将一个路由配置嵌套在另一个路由配置中，形成路由嵌套的结构。这种嵌套结构可以让我们更好地组织和管理我们的应用程序路由。

在 Vue Router 4 中，我们可以使用嵌套路由映射来实现这个功能。下面是一个示例，演示如何使用嵌套路由映射：

import { createRouter, createWebHistory } from 'vue-router'
import Home from './views/Home.vue'
import About from './views/About.vue'
import User from './views/User.vue'
import UserProfile from './views/UserProfile.vue'
import UserPosts from './views/UserPosts.vue'

const routes = [
  {
    path: '/',
    name: 'home',
    component: Home
  },
  {
    path: '/about',
    name: 'about',
    component: About
  },
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: '',
        name: 'user',
        component: UserProfile
      },
      {
        path: 'posts',
        name: 'userPosts',
        component: UserPosts
      }
    ]
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router

在这个示例中，我们定义了一个嵌套路由映射，其中一个路由（/user/:id）嵌套了两个子路由。当我们访问 /user/:id 路径时，将会渲染 User 组件。User 组件中还有两个 <router-view> 占位符，分别用于渲染 UserProfile 和 UserPosts 组件。

其中，:id 是一个动态路由参数，它可以匹配一个用户的 ID。当我们访问 /user/123 路径时，将会渲染 User 组件，并将路由参数 id 的值设置为 123。

在子路由中，我们使用 path 来定义子路由的路径。当访问 /user/123 路径时，会先匹配父路由 /user/:id，然后再匹配子路由。例如，当访问 /user/123/posts 路径时，将会渲染 UserPosts 组件，并将路由参数 id 的值设置为 123。

总之，嵌套路由映射可以帮助我们更好地组织和管理应用程序的路由。如果你的应用程序具有多层次的页面结构，可以考虑使用嵌套路由映射来管理路由。

除了基本的嵌套路由映射，Vue Router 4 还提供了一些高级用法，以帮助我们更好地组织和管理应用程序路由。

# 命名视图

命名视图是一种在单个组件中多次使用 <router-view> 的方法，它允许我们在同一个组件中同时渲染多个组件。命名视图的语法如下：

<router-view name="view1"></router-view>
<router-view name="view2"></router-view>

在路由配置中，我们可以使用 components 属性来定义命名视图。如下示例所示：

const routes = [
  {
    path: '/',
    components: {
      default: Home,
      view1: Foo,
      view2: Bar
    }
  }
]

在这个示例中，我们在根路由配置中定义了三个组件：默认组件 Home，以及两个命名视图组件 Foo 和 Bar。当我们访问根路径时，将会渲染这三个组件。其中，default 表示默认视图，如果没有给 <router-view> 指定名称，则会使用默认视图。

在嵌套路由中，我们也可以使用命名视图来渲染多个组件。例如：

const routes = [
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: '',
        components: {
          default: UserProfile,
          view1: UserAvatar,
          view2: UserBio
        }
      }
    ]
  }
]

在这个示例中，我们在 User 组件中使用了三个命名视图组件：默认组件 UserProfile，以及两个命名视图组件 UserAvatar 和 UserBio。

# 路由别名

路由别名是指将一个路径映射到另一个路径的方法。它允许我们使用不同的路径访问同一个路由，这对于重构和调整路由结构非常有用。

在路由配置中，我们可以使用 alias 属性来定义路由别名。例如：

const routes = [
  {
    path: '/about',
    component: About,
    alias: '/about-us'
  }
]

在这个示例中，我们将 /about 路径映射到 /about-us 路径上。当我们访问 /about 或 /about-us 路径时，都将会渲染 About 组件。

在嵌套路由中，我们也可以使用路由别名。例如：

const routes = [
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: 'profile',
        components: {
          default: UserProfile,
          view1: UserAvatar,
          view2: UserBio
        }
      },
      {
        path: 'settings',
        component: UserSettings,
        alias: 'preferences'
      }
    ]
  }
]

在这个示例中，我们将 /user/:id/settings 路径映射到 /user/:id/preferences 路径上。当我们 访问 /user/:id/settings 或 /user/:id/preferences 路径时，都将会渲染 UserSettings 组件。

# 路由元信息

路由元信息是指在路由配置中添加自定义数据的方法，它可以用于实现登录验证、路由拦截等功能。

在路由配置中，我们可以使用 meta 属性来定义路由元信息。例如：

const routes = [
  {
    path: '/dashboard',
    component: Dashboard,
    meta: {
      requiresAuth: true
    }
  }
]

在这个示例中，我们为 /dashboard 路由添加了一个 requiresAuth 属性，用于表示该路由需要登录验证。在路由导航时，我们可以通过访问路由对象的 meta 属性来获取该元信息：

router.beforeEach((to, from, next) => {
  if (to.meta.requiresAuth && !isAuthenticated()) {
    next('/login')
  } else {
    next()
  }
})

在这个示例中，我们使用 beforeEach 路由钩子函数来拦截路由导航，判断目标路由是否需要登录验证，如果需要且用户未登录，则跳转到登录页面。

在嵌套路由中，我们也可以为子路由添加元信息。例如：

const routes = [
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: '',
        components: {
          default: UserProfile,
          view1: UserAvatar,
          view2: UserBio
        },
        meta: {
          requiresAuth: true
        }
      }
    ]
  }
]

在这个示例中，我们为 /user/:id 路由的子路由添加了一个 requiresAuth 属性，用于表示该子路由需要登录验证。在路由导航时，如果目标路由的任一子路由需要登录验证，则进行相应判断和跳转。

# 动态路由匹配

在 Vue Router 中，我们还可以使用动态路由匹配来实现更加灵活的路由映射。

动态路由匹配是指使用带参数的路由路径，例如 /user/:id，其中 :id 表示一个动态参数，该参数可以在路由组件中通过 $route.params.id 获取。

在路由配置中，我们可以使用冒号（:）来定义动态参数。例如：

const routes = [
  {
    path: '/user/:id',
    component: UserProfile
  }
]

在这个示例中，我们定义了一个带有动态参数 id 的路由 /user/:id，该路由将匹配所有以 /user/ 开头的路径，并将匹配到的动态参数传递给 UserProfile 组件。

在路由组件中，我们可以使用 $route.params 对象来访问路由参数。例如：

// UserProfile.vue
export default {
  props: ['id'],
  created() {
    console.log(this.id) // 访问路由参数
  }
}

在这个示例中，我们使用了组件的 props 属性来声明一个 id 属性，该属性将通过路由参数传递给组件。在组件的 created 钩子函数中，我们访问了该属性，从而获得了动态参数的值。

在嵌套路由中，我们也可以使用动态路由匹配。例如：

const routes = [
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: 'profile',
        component: UserProfile
      }
    ]
  }
]

在这个示例中，我们定义了一个带有动态参数 id 的路由 /user/:id，该路由下还包含一个子路由 /profile。当访问 /user/123/profile 路径时，该路径将会匹配到 /user/:id 路由，并将动态参数 id 的值设为 123，然后再匹配到子路由 /profile。

在 UserProfile 组件中，我们同样可以通过 $route.params 对象来访问路由参数：

// UserProfile.vue
export default {
  props: ['id'],
  created() {
    console.log(this.id) // 访问路由参数
  }
}

在这个示例中，我们同样使用了组件的 props 属性来声明一个 id 属性，该属性将通过路由参数传递给组件。在组件的 created 钩子函数中，我们访问了该属性，从而获得了动态参数的值。

动态路由匹配可以帮助我们更加灵活地配置路由，实现不同场景下的路由映射。例如，我们可以使用动态路由匹配来实现用户个人主页的访问：

const routes = [
  {
    path: '/user/:id',
    component: User,
    children: [
      {
        path: '',
        component: UserProfile
      },
      {
        path: 'posts',
        component: UserPosts
      },
      {
        path: 'settings',
        component: UserSettings
      }
    ]
  }
]

在这个示例中，我们定义了一个 /user/:id 路由，该路由下包含三个子路由，分别对应用户个人主页、用户文章列表和用户设置页面。

当访问 /user/123 路径时，该路径将匹配到 /user/:id 路由，并将动态参数 id 的值设为 123，然后再匹配到 UserProfile 组件。

当访问 /user/123/posts 路径时，该路径将匹配到 /user/:id 路由，并将动态参数 id 的值设为 123，然后再匹配到 UserPosts 组件。

当访问 /user/123/settings 路径时，该路径将匹配到 /user/:id 路由，并将动态参数 id 的值设为 123，然后再匹配到 UserSettings 组件。

通过这样的路由配置，我们可以实现不同场景下的路由映射，提供更加灵活的路由访问方式。

# 命名路由

在 Vue Router 中，我们还可以使用命名路由来实现更加直观的路由访问方式。

命名路由是指给路由配置一个名字，然后通过这个名字来访问路由。在路由配置中，我们可以使用 name 属性来定义命名路由。例如：

const routes = [
  {
    path: '/',
    name: 'home',
    component: Home
  },
  {
    path: '/about',
    name: 'about',
    component: About
  }
]

在这个示例中，我们定义了两个路由，分别对应根路径和 /about 路径。这两个路由都使用了 name 属性来定义命名路由。

在代码中，我们可以使用 $router.push 方法来访问命名路由。例如：

// 在组件中跳转到 home 路由
this.$router.push({ name: 'home' })

// 在组件中跳转到 about 路由
this.$router.push({ name: 'about' })

通过这样的方式，我们可以直接使用路由的名字来访问路由，而不必使用路由的路径。这种方式能够使代码更加简洁易懂，提高开发效率。

# 命名视图

在 Vue Router 中，我们还可以使用命名视图来实现更加灵活的组件渲染方式。

命名视图是指给路由组件中的多个视图区域命名，然后通过这些命名来控制组件的渲染。在路由配置中，我们可以使用 components 属性来定义命名视图。例如：

const routes = [
  {
    path: '/',
    components: {
      default: Home,
      sidebar: Sidebar,
      footer: Footer
    }
  },
  {
    path: '/about',
    components: {
      default: About,
      sidebar: Sidebar,
      footer: Footer
    }
  }
]

在这个示例中，我们定义了两个路由，分别对应根路径和 /about 路径。这两个路由都使用了 components 属性来定义命名视图。

在组件中，我们可以使用 <router-view> 标签来指定路由组件中的视图渲染区域。在默认情况下，<router-view> 标签会渲染默认视图（即没有命名的视图）。如果我们想渲染命名视图，可以使用 name 属性来指定要渲染的视图名称。例如：

<!-- 渲染默认视图 -->
<router-view></router-view>

<!-- 渲染 sidebar 视图 -->
<router-view name="sidebar"></router-view>

<!-- 渲染 footer 视图 -->
<router-view name="footer"></router-view>

通过这样的方式，我们可以灵活地控制路由组件中的视图渲染，实现更加复杂的页面布局效果。

# 总结

总结

Vue Router 4 中的嵌套路由、动态路由匹配、命名路由和命名视图等特性，为我们实现复杂的页面路由提供了很大的灵活性和可扩展性。在实际开发中，我们可以根据具体的业务需求来选择不同的路由配置方式，提高开发效率和代码可读性。
