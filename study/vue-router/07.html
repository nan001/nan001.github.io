# vue-router4 细致的导航控制

在 Vue Router 4 中，提供了一些细致的导航控制方式，让我们可以更加灵活地控制路由的导航。下面是一些常用的导航控制方式：

# 1、 router.push()

router.push() 方法用于在代码中进行路由导航，它接受一个 Location 对象作为参数，可以包含路由的路径、参数、查询参数等信息。例如：

// 在代码中进行路由导航
router.push({ path: '/home', query: { name: 'Alice' }})

# 2、 router.replace()

router.replace() 方法和 router.push() 方法类似，也用于在代码中进行路由导航，不同之处在于 router.replace() 不会添加新的历史记录，而是替换当前的历史记录。例如：

// 在代码中进行路由导航，替换当前的历史记录
router.replace('/home')

# 3、router.go()

router.go() 方法用于在浏览器的历史记录中前进或后退多少步。例如：

// 在浏览器的历史记录中后退一步
router.go(-1)

// 在浏览器的历史记录中前进一步
router.go(1)

# 4、router.back()

router.back() 方法用于在浏览器的历史记录中后退一步。例如：

// 在浏览器的历史记录中后退一步
router.back()

# 5、router.forward()

router.forward() 方法用于在浏览器的历史记录中前进一步。例如：

// 在浏览器的历史记录中前进一步
router.forward()

# 6、router.beforeEach()

router.beforeEach() 方法用于在路由导航开始前进行全局的前置守卫。在 router.beforeEach() 中可以通过调用 next() 方法来继续导航，或者通过传递一个路由对象来取消导航。例如：

// 注册全局的前置守卫
router.beforeEach((to, from, next) => {
  // 判断用户是否已经登录
  if (isAuthenticated()) {
    // 继续导航
    next()
  } else {
    // 取消导航，并跳转到登录页面
    next('/login')
  }
})

# 7、router.beforeResolve()

router.beforeResolve() 方法用于在路由导航开始前进行全局的解析守卫。在 router.beforeResolve() 中可以通过调用 next() 方法来继续导航，或者通过传递一个路由对象来取消导航。例如：

// 注册全局的解析守卫
router.beforeResolve((to, from, next) => {
  // 在导航开始前进行异步数据的加载
  loadDataAsync().then(() => {
    // 继续导航
    next()
  })
})

可以使用 beforeRouteUpdate 钩子来检测在当前路由改变时，与当前路由有关的组件是否需要更新。这个钩子函数接收三个参数：to、from 和 next，分别表示将要到达的路由、当前路由和继续执行的函数。在该钩子函数中，你可以检测路由参数或查询参数的变化，从而决定是否需要更新组件。

例如，假设我们有一个路由 /user/:id，其中 :id 表示用户 ID，当我们从 /user/1 跳转到 /user/2 时，需要重新获取用户数据来更新页面。此时我们可以在组件中定义 beforeRouteUpdate 钩子函数：

export default {
  data() {
    return {
      user: null
    }
  },
  methods: {
    fetchUser(id) {
      // 发送请求获取用户数据
      // ...
    }
  },
  beforeRouteUpdate(to, from, next) {
    // 如果路由参数发生了变化，重新获取用户数据
    if (to.params.id !== from.params.id) {
      this.fetchUser(to.params.id)
    }
    next()
  },
  mounted() {
    // 组件第一次加载时获取用户数据
    this.fetchUser(this.$route.params.id)
  }
}

在这个示例中，我们在 beforeRouteUpdate 钩子函数中检测路由参数的变化，并根据变化重新获取用户数据来更新页面。同时，在组件第一次加载时，我们也需要调用 fetchUser 方法来获取用户数据。

除了 beforeRouteUpdate 钩子函数外，还有其他几个钩子函数可以用来细致地控制导航行为，例如 beforeRouteLeave、beforeRouteEnter 和 beforeEach 等。这些钩子函数的具体用法可以参考官方文档。

当使用导航守卫的 beforeRouteEnter 钩子时，如果需要访问组件实例（this）时会遇到一个问题，因为组件实例在这个钩子函数执行之前是不可用的。这时候可以使用一个回调函数来获取组件实例。例如：

beforeRouteEnter (to, from, next) {
  next(vm => {
    // 通过 `vm` 访问组件实例
  })
}

类似的，当使用导航守卫的 beforeRouteUpdate 钩子时，也需要使用一个回调函数来获取组件实例。例如：

beforeRouteUpdate (to, from, next) {
  // `this` 不能在这里使用，因为组件实例还没有更新
  next(vm => {
    // 通过 `vm` 访问组件实例
  })
}

在导航守卫的 beforeRouteLeave 钩子中，可以使用 next(false) 来取消当前的导航。例如：

beforeRouteLeave (to, from, next) {
  if (someCondition) {
    next(false)
  } else {
    next()
  }
}

除了使用 next(false) 取消导航之外，还可以使用 next('/path') 或 next({ name: 'name', params: { id: 123 }}) 来重定向到一个新的路径。例如：

beforeRouteLeave (to, from, next) {
  if (someCondition) {
    next('/home')
  } else {
    next()
  }
}
