# vue-router4 动态路由、方法

# 动态路由

Vue Router 4 中的动态路由可以让我们根据不同的 URL 动态加载不同的组件和数据，非常灵活和方便。在使用动态路由时，我们可以使用动态路由参数，也可以使用正则表达式来匹配路由。

# 动态路由参数

动态路由参数可以让我们根据不同的 URL，动态地加载不同的组件或者数据。例如，我们可以将路由地址设为 /user/:id，其中 :id 就是动态路由参数，表示一个动态的用户 id。这个路由地址匹配的所有 URL 都将被重定向到同一个路由，只是动态路由参数不同。

在组件中，可以通过 $route.params 访问动态路由参数。例如，在上面的例子中，可以通过 $route.params.id 访问路由参数。

我们可以在路由配置中使用动态路由参数来定义动态路由。例如：

const router = createRouter({
  routes: [
    {
      path: '/user/:id',
      component: User,
    },
  ],
});

# 正则表达式路由

除了使用动态路由参数，Vue Router 4 还支持使用正则表达式来匹配路由。例如，我们可以使用正则表达式来匹配以 /user 开头，后面跟着任意数字的 URL。这可以使用如下代码来实现：

const router = createRouter({
  routes: [
    {
      path: /^\/user\/\d+$/,
      component: User,
    },
  ],
});

在这个例子中，路由地址的 path 属性使用了一个正则表达式来匹配 URL，这个正则表达式可以匹配以 /user 开头，后面跟着任意数字的 URL。当 URL 匹配成功时，就会加载 User 组件。

值得注意的是，使用正则表达式来匹配路由可能会使路由配置变得复杂，也可能会对性能造成一定的影响。因此，应该根据具体的需求来决定是否使用正则表达式路由。

除了使用正则表达式来匹配路由，Vue Router 4 还支持使用自定义的匹配函数来匹配路由。这个函数接收两个参数：路由对象和路由地址。如果函数返回 true，则表示 URL 匹配成功，将会加载相应的组件。

# 定义动态路由

在定义动态路由时，需要在路由路径中使用参数。这些参数用冒号 : 开头，后面跟着参数名称。例如，如果我们想要匹配路径 /users/:id，我们可以使用以下路由定义：

const routes = [
  {
    path: '/users/:id',
    name: 'User',
    component: User
  }
]

在上面的示例中，我们使用了 :id 参数，表示该部分路径可以匹配任何值。

# 获取动态路由参数

在组件中使用动态路由时，我们可以通过 $route.params 对象来获取参数。例如，对于上面的路由定义，我们可以使用以下代码来获取路由参数：

export default {
  mounted() {
    console.log(this.$route.params.id) // 输出路由参数 id 的值
  }
}

在上面的示例中，我们通过 $route.params.id 来访问路由参数 id 的值。

# 监听动态路由参数的变化

如果你想在路由参数发生变化时更新组件，你可以使用 watch 来监听 $route 对象的变化。例如，下面的代码将监听 $route.params.id 的变化，并在参数变化时更新组件：

export default {
  watch: {
    '$route.params.id'(newVal, oldVal) {
      // 参数变化时的操作
    }
  }
}

# 动态路由的懒加载

在 Vue Router 4 中，可以使用 import() 函数来实现动态路由的懒加载。这种方法可以帮助你提高应用程序的性能，因为它可以延迟加载组件代码，而不是在应用程序加载时全部加载。以下是一个使用 import() 函数进行懒加载的示例：

const routes = [
  {
    path: '/users/:id',
    name: 'User',
    component: () => import('./components/User.vue')
  }
]

在上面的示例中，我们将 component 属性设置为一个函数，这个函数会在路由被访问时异步加载组件。当路由被访问时，Vue Router 4 将自动使用 Webpack 进行代码分割，并将组件代码作为单独的文件加载。

在使用动态路由时，我们还需要注意以下几点：

动态路由参数的变化会重新触发组件的创建和销毁，因此需要在组件内部正确地处理这些变化。 动态路由参数的匹配顺序与路由规则的顺序有关，因此需要注意路由规则的编写顺序。 动态路由参数可以通过正则表达式进行匹配，从而更加灵活地匹配不同的参数值。 动态路由参数也可以在编程式导航和路由链接中使用，从而实现更加灵活的路由跳转。

# 动态路由方法

动态路由主要是通过这两个函数来实现的，其中 router.addRoute() 用于动态添加路由，router.removeRoute() 用于动态移除路由。

使用这两个函数，我们可以在路由运行时添加和移除路由。这意味着我们可以根据应用程序的需求来动态地创建和销毁路由。比如在某些场景下，我们需要动态地根据后端返回的数据来生成路由，而这时候就可以使用动态路由来实现。

需要注意的是，添加的路由只有在下一次路由跳转时才会生效，而不会立即生效。这是因为在路由注册时，Vue Router 会先遍历一遍所有的路由规则，所以新添加的路由必须要等到下一次遍历时才会被生效。

另外，使用动态路由时需要特别小心，因为动态路由可能会带来潜在的安全风险。如果你不对动态路由做出正确的处理，可能会导致应用程序出现安全漏洞，比如跨站脚本攻击。因此，在使用动态路由时一定要谨慎，并对用户输入进行严格的验证和过滤。

# router.addRoute(route: RouteConfig)：添加路由

该方法用于向路由表中动态添加一条路由记录。需要注意的是，该方法只能在路由初始化之后调用。

示例：

const newRoute = {
  path: '/new-route',
  component: NewComponent
}
router.addRoute(newRoute)

# router.removeRoute(name: string)：移除路由

该方法用于从路由表中移除一条路由记录。需要注意的是，该方法只能在路由初始化之后调用。

示例：

router.removeRoute('name-of-route-to-be-removed')

# router.hasRoute(location: RawLocation | RouteLocation)：判断路由是否存在

该方法用于判断某个路由是否存在于路由表中。

示例：

const exists = router.hasRoute('/some-route')

# router.getRoutes()：获取当前所有路由

该方法用于获取当前路由表中的所有路由记录。

示例：

const allRoutes = router.getRoutes()

# router.resolve(location: RawLocation, current?: Route, append?: boolean): Promise<Resolved>

该方法用于解析路由并返回一个 Promise，该 Promise 将返回包含当前路由信息的对象。如果解析失败，则会被拒绝并返回错误对象。

示例：

router.resolve('/some-route').then((resolved) => {
  console.log(resolved)
}).catch((error) => {
  console.error(error)
})

# router.beforeResolve((to, from, next) => {})：在路由解析之前执行

该方法用于在路由解析之前执行一些逻辑，比如获取用户信息、权限验证等。

示例：

router.beforeResolve((to, from, next) => {
  // 获取用户信息并保存到 store 中
  store.dispatch('getUserInfo').then(() => {
    next()
  }).catch(() => {
    next('/login')
  })
})

# router.beforeEach((to, from, next) => {})：在路由跳转之前执行

该方法用于在路由跳转之前执行一些逻辑，比如判断用户是否登录等。

示例：

router.beforeEach((to, from, next) => {
  if (to.meta.requiresAuth && !store.state.isAuthenticated) {
    next('/login')
  } else {
    next()
  }
})

# router.afterEach((to, from) => {})：在路由跳转之后执行

该方法用于在路由跳转之后执行一些逻辑，比如上报 PV 数据等。

示例：

router.afterEach((to, from) => {
  // 上报 PV 数据
  reportPV(to.path)
})

以上是 Vue Router 4 中常用的动态路由方法，使用这些方法可以实现更加灵活和高效的路由管理。

在使用 router.addRoute() 方法添加动态路由时，我们需要提供一个路由配置对象，这个对象包含了路由的各种属性，例如：

router.addRoute('parent', {
  path: '/child',
  component: ChildComponent
})

在这个例子中，我们为名为 parent 的路由添加了一个子路由 child，子路由的组件是 ChildComponent。在这个路由配置对象中，还可以包含许多其他属性，例如 meta 和 props 等，这些属性的作用和使用方式与常规路由的配置是一样的。

在使用 router.removeRoute() 方法删除动态路由时，我们需要提供要删除的路由名称或路由配置对象。例如，我们可以使用以下代码从路由器中删除名为 parent 的路由：

router.removeRoute('parent')

或者，我们也可以通过提供路由配置对象来删除路由：

const route = router.getRoutes().find(route => route.name === 'parent')
router.removeRoute(route)

在这个例子中，我们使用 router.getRoutes() 方法获取路由器中的所有路由，然后从中查找名称为 parent 的路由，最后将这个路由对象传递给 router.removeRoute() 方法来删除路由。

需要注意的是，当我们使用 router.removeRoute() 方法删除一个路由时，它会删除整个路由记录，包括该路由下的所有子路由。如果我们只想删除某个子路由，可以先通过 router.getRoutes() 方法获取到该子路由的父路由对象，然后在父路由对象上使用 children 属性删除子路由。

例如，假设我们要删除父路由为 parent 的路由下名为 child 的子路由，可以使用以下代码：

const parentRoute = router.options.routes.find(route => route.name === 'parent')
const childIndex = parentRoute.children.findIndex(route => route.name === 'child')
parentRoute.children.splice(childIndex, 1)
router.removeRoute('parent-child')

这里首先通过 router.options.routes 找到父路由为 parent 的路由配置，然后再从其中的 children 数组中找到名为 child 的子路由，最后使用 splice() 方法将其从 children 数组中删除。需要注意的是，这里删除的是路由配置，而不是路由实例，因此我们还需要通过 router.removeRoute() 方法将其从路由表中删除。

# demo

下面是一个使用addRoute和removeRoute实现动态路由的示例：

import { createRouter, createWebHistory } from 'vue-router'

const router = createRouter({
  history: createWebHistory(),
  routes: [
    {
      path: '/',
      component: () => import('./views/Home.vue')
    },
    {
      path: '/parent/:id',
      component: () => import('./views/Parent.vue'),
      children: [
        {
          path: 'child/:childId',
          component: () => import('./views/Child.vue')
        }
      ]
    }
  ]
})

// 添加路由
router.addRoute('parent', {
  path: '/parent/new-child',
  component: () => import('./views/NewChild.vue')
})

// 删除路由
router.removeRoute('parent-child')

export default router

上面的示例中，我们首先创建一个基本的Vue Router实例，并定义了一个带有动态路由的初始路由配置。然后，我们使用addRoute方法添加了一个名为parent的新路由，该路由将作为该路由的子路由。最后，我们使用removeRoute方法删除了名为parent-child的路由。
