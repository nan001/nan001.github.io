# Promise 封装

防抖和节流都是前端开发中常用的优化技术，用于限制事件触发的频率，提高页面性能和用户体验。

防抖：指触发事件后，等待一段时间后才执行处理函数。如果在等待时间内再次触发该事件，则重新计时。防抖主要用于限制频繁触发的事件，如窗口大小变化、输入框输入等，以减少事件处理函数的执行次数。例如，在搜索框输入时，我们可以使用防抖来限制搜索请求的发送频率，避免发送大量无用的网络请求。

节流：指在一定时间间隔内只触发一次事件处理函数。如果在该时间间隔内触发了多次事件，则只执行第一次事件的处理函数，忽略其余的事件。节流主要用于限制事件处理函数的执行次数，尤其适用于一些高频事件，如滚动、鼠标移动等。例如，在滚动到底部时，我们可以使用节流来限制触发事件的处理函数，避免过多的计算和渲染操作。

总的来说，防抖和节流都是用来优化事件处理的技术，可以减少事件处理函数的执行次数，提高页面性能和用户体验。防抖和节流的具体实现可以使用定时器、闭包等方式，也可以使用一些库或框架提供的封装函数，如Lodash库中的debounce()和throttle()函数。

# Lodash库中的debounce()和throttle()函数是两个常用的防抖和节流函数。

debounce()函数用于防抖，它接受两个参数：一个是要执行的函数，另一个是等待时间（单位毫秒）。当调用debounce()函数时，它会返回一个新的函数，这个新函数会在等待时间内被多次调用，但只有最后一次调用会触发执行原函数。如果在等待时间内再次调用了这个新函数，则会重新开始计时。这个函数可以用于限制一些频繁触发的事件，如窗口大小变化、输入框输入等。

# 例如，下面的代码使用debounce()函数实现了一个输入框防抖的例子：

import debounce from 'lodash/debounce';

const searchInput = document.getElementById('search-input');
const searchUsers = debounce(() => {
  // 发送搜索请求
}, 500);

searchInput.addEventListener('input', () => {
  searchUsers();
});

在这个例子中，当输入框输入时，searchUsers()函数会被调用，但由于使用了debounce()函数，它只会在输入停止500毫秒后才真正执行，避免了频繁的网络请求。

throttle()函数用于节流，它接受两个参数：一个是要执行的函数，另一个是时间间隔（单位毫秒）。当调用throttle()函数时，它会返回一个新的函数，这个新函数在时间间隔内只能被执行一次。如果在时间间隔内再次调用了这个新函数，则会被忽略。这个函数可以用于限制一些高频事件的处理，如滚动、鼠标移动等。

# 例如，下面的代码使用throttle()函数实现了一个滚动事件节流的例子：

import throttle from 'lodash/throttle';

const scrollContainer = document.getElementById('scroll-container');
const handleScroll = throttle(() => {
  // 处理滚动事件
}, 100);

scrollContainer.addEventListener('scroll', handleScroll);

在这个例子中，当滚动容器发生滚动时，handleScroll()函数会被调用，但由于使用了throttle()函数，它只会在100毫秒内被执行一次，避免了过多的计算和渲染操作。

# Promise 封装防抖示例

下面是一个完整的使用Promise对象封装的防抖函数示例：

function debounce(func, delay) {
  let timeoutId;
  return function (...args) {
    return new Promise((resolve, reject) => {
      clearTimeout(timeoutId);
      timeoutId = setTimeout(() => {
        try {
          const result = func.apply(this, args);
          resolve(result);
        } catch (error) {
          reject(error);
        }
      }, delay);
    });
  };
}

function fetchData(keyword) {
  return fetch(`https://api.github.com/search/repositories?q=${keyword}`)
    .then((response) => {
      if (!response.ok) {
        throw new Error(`HTTP error ${response.status}`);
      }
      return response.json();
    })
    .then((data) => {
      const results = data.items.slice(0, 5).map((item) => ({
        name: item.name,
        url: item.html_url,
        stars: item.stargazers_count,
        description: item.description,
      }));
      return results;
    })
    .catch((error) => {
      console.error(error);
    });
}

const searchInput = document.querySelector('#search-input');
const searchResults = document.querySelector('#search-results');

searchInput.addEventListener('input', debounce((event) => {
  const keyword = event.target.value.trim();
  if (keyword) {
    fetchData(keyword)
      .then((results) => {
        searchResults.innerHTML = results.map((result) => `
          <li>
            <h3><a href="${result.url}">${result.name}</a></h3>
            <p>${result.description}</p>
            <p>Stars: ${result.stars}</p>
          </li>
        `).join('');
      })
      .catch((error) => console.error(error));
  } else {
    searchResults.innerHTML = '';
  }
}, 500));

这个例子中，我们定义了一个名为debounce()的防抖函数，它接受两个参数：要执行的函数和防抖延迟时间。当debounce()函数被调用时，它返回一个新的函数，该函数将使用Promise对象来封装要执行的函数，并实现防抖功能。每当返回的函数被调用时，它将清除上一个计时器、设置一个新的计时器，并返回一个Promise对象。当计时器时间到达防抖延迟时间时，该Promise对象将被解析，并执行封装的函数，并返回执行结果；如果执行过程中发生错误，则该Promise对象将被拒绝，并输出错误信息。

在这个例子中，我们使用fetchData()函数来获取Github上与搜索关键字相关的前5个仓库信息。每当搜索输入框发生输入事件时，我们将搜索关键字传递给debounce()函数，并在防抖延迟时间后执行fetchData()函数。当fetchData()函数成功执行时，我们将搜索结果渲染到页面上；如果fetchData()函数执行失败，则输出错误信息。通过这种方式，我们可以使用Promise对象来封装异步操作，并实现防抖功能，从而提高代码的可读性和可维护性。

# Promise 封装节流示例

以下是一个完整的axios和Promise封装的节流示例代码：

// 引入axios库
import axios from 'axios'

// 搜索Github用户信息的函数
function searchUsers(keyword) {
  return axios.get(`https://api.github.com/search/users?q=${keyword}&per_page=5`)
    .then(response => response.data.items)
}

// 封装一个节流函数
function throttle(fn, delay) {
  let lastTime = 0
  return function(...args) {
    const now = Date.now()
    if (now - lastTime >= delay) {
      fn.apply(this, args)
      lastTime = now
    }
  }
}

// 获取搜索输入框和展示结果的DOM元素
const searchInput = document.querySelector('#searchInput')
const searchResult = document.querySelector('#searchResult')

// 使用节流函数封装searchUsers()函数
const throttleSearchUsers = throttle(function(keyword) {
  searchUsers(keyword)
    .then(users => {
      // 清空搜索结果列表
      searchResult.innerHTML = ''
      // 显示用户信息
      users.forEach(user => {
        const li = document.createElement('li')
        li.innerText = user.login
        searchResult.appendChild(li)
      })
    })
    .catch(error => console.log(error))
}, 500)

// 监听搜索输入框的输入事件
searchInput.addEventListener('input', function(event) {
  const keyword = event.target.value.trim()
  if (keyword) {
    throttleSearchUsers(keyword)
  } else {
    // 清空搜索结果列表
    searchResult.innerHTML = ''
  }
})

在这个示例中，我们使用axios库发送HTTP请求获取Github用户信息，然后使用Promise对searchUsers()函数进行封装。在使用searchUsers()函数时，我们使用封装好的throttle函数对其进行了节流，以限制请求发送的频率。最后，我们监听搜索输入框的输入事件，并使用节流后的searchUsers()函数来获取用户信息，并展示在页面上。

# Promise 封装 axios 请求 token 失效无感刷新

在前端开发中，经常会使用 Axios 库来进行 HTTP 请求，而大多数接口都需要携带用户的 token 进行鉴权。但是，如果用户的 token 过期了，那么所有的接口请求都将会失败。在这种情况下，我们希望能够自动地刷新用户的 token 并重新发送之前的请求，使得用户不会因为 token 失效而感到不便。

下面是一个使用 Promise 封装 Axios 请求，并处理 token 失效的示例代码：

import axios from 'axios';

// 创建一个 Axios 实例
const instance = axios.create({
  baseURL: 'http://example.com/api',
  timeout: 5000,
});

// 刷新 token 的方法
function refreshToken() {
  // 发送请求，获取新的 token
  return instance.post('/refresh_token', { refreshToken: 'xxx' }).then((res) => {
    // 将新的 token 存储到本地存储中
    localStorage.setItem('token', res.data.token);
    localStorage.setItem('refreshToken', res.data.refreshToken);
    // 返回新的 token
    return res.data.token;
  });
}

// 在请求拦截器中添加 token
instance.interceptors.request.use(
  (config) => {
    // 从本地存储中获取 token
    const token = localStorage.getItem('token');
    // 如果存在 token，则将其添加到请求头中
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => {
    return Promise.reject(error);
  }
);

// 在响应拦截器中处理 token 失效
instance.interceptors.response.use(
  (response) => {
    return response;
  },
  (error) => {
    const originalRequest = error.config;
    // 如果请求返回的状态码为 401，则说明 token 失效
    if (error.response.status === 401 && !originalRequest._retry) {
      originalRequest._retry = true;
      // 发送刷新 token 的请求
      return refreshToken().then((token) => {
        // 将新的 token 添加到请求头中
        instance.defaults.headers.common['Authorization'] = 'Bearer ' + token;
        // 将原始请求重新发送
        return instance(originalRequest);
      });
    }
    return Promise.reject(error);
  }
);

// 封装一个请求方法
function request(url, method, params) {
  return instance({
    url,
    method,
    params,
  });
}

export default request;

在上面的代码中，我们首先创建了一个 Axios 实例，并在请求拦截器中添加了 token，用于进行鉴权。在响应拦截器中，我们判断了请求返回的状态码，如果是 401，则说明用户的 token 失效了，需要进行刷新。我们通过调用 refreshToken() 方法来获取新的 token，然后将其存储到本地存储中，并将其添加到请求头中重新发送之前的请求。

最后，我们封装了一个 request() 方法，用于发起 HTTP 请求。在调用该方法时，只需要传入请求的 URL、请求方法以及请求参数即可。如果用户的 token 失效，我们的代码会自动刷新 token 并重新发送请求。

下面是一个简单的示例，演示如何使用 Promise 封装 Axios 请求并自动刷新 token：

import axios from 'axios';

class ApiClient {
  constructor() {
    this.token = null;
  }

  setToken(token) {
    this.token = token;
  }

  request(url, method, params) {
    const headers = {
      Authorization: `Bearer ${this.token}`,
    };

    const config = {
      url,
      method,
      headers,
    };

    if (method === 'get') {
      config.params = params;
    } else {
      config.data = params;
    }

    return axios(config).catch((error) => {
      if (error.response && error.response.status === 401) {
        // Token expired, refresh token and try again
        return this.refreshToken().then(() => this.request(url, method, params));
      }

      throw error;
    });
  }

  refreshToken() {
    const refreshUrl = 'https://api.example.com/auth/refresh';
    const refreshConfig = {
      url: refreshUrl,
      method: 'post',
      headers: {
        Authorization: `Bearer ${this.token}`,
      },
    };

    return axios(refreshConfig).then((response) => {
      this.token = response.data.token;
    });
  }
}

const apiClient = new ApiClient();

// set token before making request
apiClient.setToken('your_access_token');

// make request with automatic token refresh
apiClient.request('/api/users', 'get', { limit: 10 }).then((response) => {
  console.log(response.data);
}).catch((error) => {
  console.error(error);
});

在这个例子中，我们创建了一个 ApiClient 类，包含了 setToken()、request() 和 refreshToken() 三个方法。setToken() 方法用于设置 token，request() 方法用于发起 HTTP 请求并自动刷新 token，refreshToken() 方法用于刷新 token。

在 request() 方法中，我们使用了 Axios 发起 HTTP 请求，并在发生 401 错误时自动刷新 token。在这种情况下，我们会先调用 refreshToken() 方法刷新 token，然后再次调用 request() 方法发起 HTTP 请求。这样就可以避免用户手动刷新 token 的操作，提高了用户体验。

总之，Promise 封装 Axios 请求并自动刷新 token 可以帮助我们简化代码，提高代码的可读性和可维护性，并且可以提高用户体验。
