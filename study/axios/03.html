# axios 实例

Axios实例允许我们为一个应用程序中的多个请求创建一个可配置的实例，这些请求具有相同的默认配置，如baseURL、headers、timeout等。

创建Axios实例的方法是使用axios.create()方法。此方法返回一个新的Axios实例，您可以使用它来发送请求。以下是一个示例：

const instance = axios.create({
  baseURL: 'https://api.example.com',
  timeout: 1000,
  headers: {'X-Custom-Header': 'foobar'}
});

instance.get('/user')
  .then(response => {
    console.log(response.data);
  })
  .catch(error => {
    console.log(error);
  });

在上面的示例中，我们使用axios.create()方法创建了一个名为instance的Axios实例。我们传递了一个配置对象作为参数，其中包含了一些默认配置，例如baseURL、timeout和headers。

在创建实例之后，我们可以使用它来发送请求。在这种情况下，我们使用instance.get()方法发送一个GET请求。当我们收到响应时，我们打印响应数据。如果发生错误，我们将打印错误对象。

当我们使用实例发送请求时，Axios会自动将实例的默认配置与请求配置合并。例如，在上面的示例中，请求的baseURL将是https://api.example.com/user，超时时间为1秒，并且包含一个自定义的X-Custom-Header请求头。

通过创建多个Axios实例，我们可以为不同的API端点或用例使用不同的配置。例如，您可以为不同的后端服务创建不同的实例，以便为每个服务使用不同的基本URL和身份验证凭据。

除了默认配置之外，Axios实例还允许我们使用拦截器。拦截器是在请求发送或响应接收之前执行的函数。我们可以使用拦截器来修改请求或响应、打印日志或执行其他任务。以下是一个示例：

const instance = axios.create({
  baseURL: 'https://api.example.com',
  timeout: 1000,
  headers: {'X-Custom-Header': 'foobar'}
});

instance.interceptors.request.use(config => {
  // 在请求发送之前，修改请求配置
  console.log('请求已发送：', config);
  return config;
}, error => {
  // 请求错误时执行
  console.log('请求错误：', error);
  return Promise.reject(error);
});

instance.interceptors.response.use(response => {
  // 在响应接收之前，修改响应数据
  console.log('响应已接收：', response);
  return response;
}, error => {
  // 响应错误时执行
  console.log('响应错误：', error);
  return Promise.reject(error);
});

instance.get('/user')
  .then(response => {
    console.log(response.data);
  })
  .catch(error => {
    console.log(error);
  });

在上面的示例中，我们使用instance.interceptors.request.use()方法添加了一个请求拦截器。该拦截器在每个请求发送之前执行，并将请求配置打印到控制台。我们还使用instance.interceptors.response.use()方法添加了一个响应拦截器。该拦截器在每个响应接收之前执行，并将响应数据打印到控制台。

拦截器可以在请求或响应上执行任何操作，但是需要记住，拦截器在所有请求和响应上都会执行，因此应该避免在拦截器中执行重复的操作。

除了请求和响应拦截器之外，Axios实例还支持错误拦截器。错误拦截器在请求或响应中出现错误时执行。以下是一个示例：

const instance = axios.create({
  baseURL: 'https://api.example.com',
  timeout: 1000,
  headers: {'X-Custom-Header': 'foobar'}
});

instance.interceptors.response.use(response => {
  // 在响应接收之前，修改响应数据
  console.log('响应已接收：', response);
  return response;
}, error => {
  // 响应错误时执行
  console.log('响应错误：', error);

  // 判断响应错误的类型
  if (error.response) {
    console.log('服务器响应错误：', error.response.data);
  } else if (error.request) {
    console.log('请求未收到响应：', error.request);
  } else {
    console.log('请求错误：', error.message);
  }

  return Promise.reject(error);
});

instance.get('/user')
  .then(response => {
    console.log(response.data);
  })
  .catch(error => {
    console.log(error);
  });

在上面的示例中，我们使用instance.interceptors.response.use()方法添加了一个响应拦截器。如果响应出现错误，我们将判断错误的类型，并将相应的错误消息打印到控制台。

Axios实例是一个非常有用的功能，可以使我们更轻松地管理应用程序中的多个请求，并为每个请求使用不同的默认配置和拦截器。

Axios实例可以使我们更轻松地管理应用程序中的多个请求，因为我们可以为每个请求使用不同的默认配置和拦截器。以下是一个示例：

// 创建一个Axios实例，为所有请求设置基本URL和默认请求头
const instance = axios.create({
  baseURL: 'https://api.example.com',
  headers: {'Content-Type': 'application/json'}
});

// 创建另一个Axios实例，为所有请求设置超时时间和默认响应类型
const instanceWithTimeout = axios.create({
  timeout: 5000,
  responseType: 'json'
});

// 发送请求到第一个实例
instance.get('/users')
  .then(response => {
    console.log(response.data);
  })
  .catch(error => {
    console.log(error);
  });

// 发送请求到第二个实例
instanceWithTimeout.get('/posts')
  .then(response => {
    console.log(response.data);
  })
  .catch(error => {
    console.log(error);
  });

在上面的示例中，我们创建了两个不同的Axios实例。第一个实例为所有请求设置了基本URL和默认请求头。第二个实例为所有请求设置了超时时间和默认响应类型。

然后，我们使用这两个实例发送了两个不同的请求。对于每个请求，Axios将自动使用相应的默认配置和拦截器。

这使得我们可以在应用程序中管理多个请求，并为每个请求使用不同的配置和拦截器。例如，我们可以为每个请求设置不同的超时时间，或者使用不同的授权标头。

除了设置默认配置和拦截器之外，我们还可以使用Axios实例来管理应用程序中的多个请求。例如，我们可以将请求拆分为多个模块，并使用不同的Axios实例来处理每个模块的请求。

以下是一个示例，演示如何在不同的模块中使用不同的Axios实例：

// userModule.js
import axios from 'axios';

// 创建一个Axios实例，用于处理用户模块的请求
const userInstance = axios.create({
  baseURL: 'https://api.example.com/users',
  headers: {'Content-Type': 'application/json'}
});

export const getUsers = () => {
  return userInstance.get('/');
};

export const getUserById = (id) => {
  return userInstance.get(`/${id}`);
};

export const createUser = (user) => {
  return userInstance.post('/', user);
};

// postModule.js
import axios from 'axios';

// 创建另一个Axios实例，用于处理帖子模块的请求
const postInstance = axios.create({
  baseURL: 'https://api.example.com/posts',
  headers: {'Content-Type': 'application/json'}
});

export const getPosts = () => {
  return postInstance.get('/');
};

export const getPostById = (id) => {
  return postInstance.get(`/${id}`);
};

export const createPost = (post) => {
  return postInstance.post('/', post);
};

在上面的示例中，我们将请求拆分为两个模块：用户模块和帖子模块。对于每个模块，我们创建了一个不同的Axios实例，并使用它来处理该模块的所有请求。

在userModule.js文件中，我们使用userInstance实例处理所有用户模块的请求。在postModule.js文件中，我们使用postInstance实例处理所有帖子模块的请求。

这种方法使得我们可以更轻松地管理应用程序中的多个请求，并使用不同的Axios实例来处理每个请求。这使得我们可以更好地组织我们的代码，并使其更易于维护和扩展。
