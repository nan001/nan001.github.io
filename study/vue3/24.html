# vue3 内置组件

Vue3 内置组件是指在 Vue3 库中已经预先定义好的一些组件，开发者可以直接使用它们而无需进行额外的组件定义。这些组件包括但不限于：

Teleport Transition TransitionGroup Suspense KeepAlive 下面我们来简要介绍一下这些内置组件的作用和用法。

# Teleport

Teleport 是一个新的 Vue3 组件，可以将一个组件的内容渲染到 DOM 树中的任意位置。在实际开发中，通常用于在 Modal、Popover、Tooltip 等组件中使用。它有两个主要的特点：

可以指定渲染到的目标元素，支持跨组件渲染。 可以渲染任意的 Vue 组件内容。 使用方法：

<teleport to="body">
  <!-- 需要渲染的内容 -->
</teleport>

其中，to 属性指定了渲染到的目标元素。

# Transition

Transition 组件是 Vue3 提供的用于实现过渡动画的组件。它可以让组件在插入、更新、移除时，自动应用一些 CSS 过渡类名，从而实现过渡动画效果。在实际开发中，可以用于实现各种动态效果，如淡入淡出、滑动、旋转等。

使用方法：

<transition name="fade">
  <!-- 需要过渡动画的内容 -->
</transition>

其中，name 属性指定了过渡动画的名称，可以通过设置不同的名称来应用不同的动画效果。需要注意的是，为了能够正确应用过渡动画，需要配合 CSS 过渡类名使用，如 .fade-enter、.fade-leave-to 等。

# TransitionGroup

TransitionGroup 组件是在 Vue2 中就已经存在的组件，在 Vue3 中仍然可以继续使用。它是用于实现多个组件的过渡动画的组件，可以让多个组件同时执行过渡动画效果。

使用方法：

<transition-group name="list" tag="ul">
  <!-- 多个需要过渡动画的组件 -->
</transition-group>

其中，name 属性指定了过渡动画的名称，tag 属性指定了要渲染的标签名。需要注意的是，需要在 CSS 中定义相应的过渡类名。

# Suspense

Suspense 组件是一个实验性的 Vue3 组件，用于在异步组件加载完成之前展示一个占位符，可以使页面的加载体验更好。它有两种用法：

作为全局组件，可以用于整个应用的异步组件加载。 作为局部组件，可以用于单个异步组件的加载。

作为全局组件，只需要在应用程序的入口处将 Suspense 组件注册为全局组件即可：

import { createApp } from 'vue'
import App from './App.vue'
import { Suspense } from 'vue'

const app = createApp(App)

app.component('Suspense', Suspense)

app.mount('#app')

注册后，就可以在任何组件中使用 Suspense 组件了。

作为局部组件，可以在单个组件中导入和使用 Suspense 组件：

import { Suspense } from 'vue'

export default {
  components: {
    Suspense
  },
  // ...
}

然后就可以在该组件的模板中使用 Suspense 组件了。

需要注意的是，当 Suspense 组件用于异步组件加载时，它必须将要加载的异步组件作为其子元素，并在异步组件加载完成前显示占位符内容。这个占位符可以使用 template 标签或其他任何 HTML 元素实现。例如：

<template>
  <Suspense>
    <template #default>
      <AsyncComponent />
    </template>
    <template #fallback>
      <div>Loading...</div>
    </template>
  </Suspense>
</template>

在这个示例中，AsyncComponent 将作为 Suspense 的默认插槽传递，如果 AsyncComponent 正在加载，则显示 Loading... 的占位符内容。

# KeepAlive

<keep-alive> 组件是 Vue.js 内置的一个抽象组件，可以用来缓存组件。

当一个组件被包裹在 <keep-alive> 中时，这个组件在切换时并不会被销毁，而是被缓存起来，等待下一次使用。

在使用 <keep-alive> 时，需要给包裹的组件设置一个 key 属性，这个 key 值会作为缓存组件的标识符。

<template>
  <div>
    <button @click="toggleComponent">Toggle Component</button>
    <keep-alive>
      <component :is="currentComponent" :key="currentComponent"></component>
    </keep-alive>
  </div>
</template>

<script>
import ComponentA from './ComponentA.vue'
import ComponentB from './ComponentB.vue'

export default {
  components: {
    ComponentA,
    ComponentB
  },
  data() {
    return {
      currentComponent: 'ComponentA'
    }
  },
  methods: {
    toggleComponent() {
      this.currentComponent = this.currentComponent === 'ComponentA' ? 'ComponentB' : 'ComponentA'
    }
  }
}
</script>

在上面的示例中，我们使用了 <keep-alive> 组件包裹了一个动态组件，并且给动态组件设置了一个 key 属性，这个 key 值会作为缓存组件的标识符。

当我们点击按钮切换组件时，组件并没有被销毁，而是被缓存了起来，等待下一次使用。

需要注意的是，被缓存的组件可能会因为内存不足等原因被销毁，当组件被销毁时，它的 beforeUnmount 和 deactivated 生命周期钩子函数会被调用，当组件被重新使用时，它的 activated 生命周期钩子函数会被调用。

此外，<keep-alive> 组件还有一个 include 属性和一个 exclude 属性，用来配置需要缓存的组件和不需要缓存的组件。

<template>
  <div>
    <button @click="toggleComponent">Toggle Component</button>
    <keep-alive :include="includedComponents" :exclude="excludedComponents">
      <component :is="currentComponent" :key="currentComponent"></component>
    </keep-alive>
  </div>
</template>

<script>
import ComponentA from './ComponentA.vue'
import ComponentB from './ComponentB.vue'

export default {
  components: {
    ComponentA,
    ComponentB
  },
  data() {
    return {
      currentComponent: 'ComponentA',
      includedComponents: ['ComponentA'],
      excludedComponents: ['ComponentB']
    }
  },
  methods: {
    toggleComponent() {
      this.currentComponent = this.currentComponent === 'ComponentA' ? 'ComponentB' : 'ComponentA'
    }
  }
}
</script>

在上面的示例中，我们使用了 include 属性和 exclude 属性来配置需要缓存的组件和不需要缓存的组件。
