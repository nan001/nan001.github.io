# vue3 逻辑复用 自定义指令

Vue3 中的自定义指令和 Vue2 中的自定义指令有所不同。在 Vue3 中，自定义指令通过 directive 函数来定义，该函数接受一个指令名称和一个指令选项对象作为参数，并返回一个包含 beforeMount、mounted、beforeUpdate、updated、beforeUnmount 以及 unmounted 生命周期钩子函数的对象。

下面是一个简单的例子，演示了如何定义一个名为 highlight 的自定义指令：

import { directive } from 'vue';

export const highlightDirective = directive((el, binding) => {
  el.style.backgroundColor = binding.value;
});

在上面的例子中，我们通过 directive 函数来定义了一个名为 highlight 的自定义指令。该指令的实现非常简单，只是在绑定元素的背景色上应用了一个指定的颜色。

使用自定义指令也非常简单，只需要在元素上添加 v-highlight 指令，并为其设置一个颜色值：

<template>
  <div v-highlight="'yellow'">
    This text will be highlighted in yellow
  </div>
</template>

在上面的例子中，我们为一个 <div> 元素添加了 v-highlight 指令，并将其颜色设置为 'yellow'。当该组件被渲染时，highlight 指令将会被调用，并将该元素的背景色设置为黄色。

除了可以在元素上直接使用自定义指令之外，我们还可以通过 app.directive 方法来在全局注册自定义指令，或者在组件的 setup 函数中使用 directive 函数来注册局部自定义指令。

总的来说，自定义指令是 Vue3 中一个非常强大的特性，可以帮助我们实现一些非常复杂的 UI 功能和交互效果。

在Vue3中，我们可以使用 directive 函数来创建自定义指令。该函数接收两个参数：指令名称和指令配置对象。指令配置对象可以包含多个钩子函数，用于处理不同的指令生命周期。

下面是一个自定义指令的示例，该指令可以在元素上添加背景颜色，并在鼠标悬停时改变背景颜色：

import { directive } from 'vue'

const hoverBgColorDirective = directive('hover-bgcolor', {
  beforeMount(el, binding) {
    el.style.backgroundColor = binding.value
    el.addEventListener('mouseenter', () => {
      el.style.backgroundColor = binding.arg
    })
    el.addEventListener('mouseleave', () => {
      el.style.backgroundColor = binding.value
    })
  },
  beforeUnmount(el, binding) {
    el.removeEventListener('mouseenter')
    el.removeEventListener('mouseleave')
  }
})

export default hoverBgColorDirective

在上面的示例中，我们首先使用 directive 函数创建了一个名为 hover-bgcolor 的自定义指令，并为其配置了 beforeMount 和 beforeUnmount 钩子函数。在 beforeMount 钩子函数中，我们首先根据指令的绑定值（即 binding.value）设置了元素的初始背景颜色，然后添加了 mouseenter 和 mouseleave 事件监听器，用于在鼠标悬停时改变元素的背景颜色。在 beforeUnmount 钩子函数中，我们移除了 mouseenter 和 mouseleave 事件监听器，以避免内存泄漏。

使用自定义指令非常简单。我们只需在需要使用该指令的元素上添加相应的指令名称即可。例如，在以下示例中，我们将自定义指令 hover-bgcolor 应用到了一个 div 元素上：

<template>
  <div v-hover-bgcolor="'red'" arg="blue">Hover me</div>
</template>

在上面的示例中，我们将指令的绑定值设置为 'red'，这将作为元素的初始背景颜色。我们还将指令的参数设置为 'blue'，这将在鼠标悬停时作为元素的背景颜色。当鼠标悬停在元素上时，元素的背景颜色将变为 'blue'，当鼠标移出元素时，元素的背景颜色将恢复为 'red'。

在指令定义时，可以使用 binding 参数来获取指令的参数和修饰符。因此，在 handleClickOutside 方法中，可以通过 binding.arg 来获取参数 foo，通过 binding.modifiers 来获取修饰符。具体实现可以参考以下代码：

const handleClickOutside = (el, binding) => {
  const { value, arg, modifiers } = binding;

  const onClick = (e) => {
    if (!el.contains(e.target) && typeof value === "function") {
      value();
    }
  };

  document.addEventListener("click", onClick);

  onUnmounted(() => {
    document.removeEventListener("click", onClick);
  });
};

export default {
  mounted: handleClickOutside,
  updated: handleClickOutside,
};

在上述代码中，我们从 binding 中解构出了 value，即指令绑定的方法，arg，即参数 foo，和 modifiers，即修饰符。然后，在 onClick 方法中，我们根据 value 的类型是否为函数，来调用指令绑定的方法。最后，我们通过 document.addEventListener 来监听全局的 click 事件，在 onUnmounted 钩子中，使用 document.removeEventListener 来取消全局事件的监听。

使用这个自定义指令时，可以在模板中通过 v-click-outside:foo.modifier 的方式来指定参数和修饰符，例如：

<div v-click-outside:foo.prevent="handleClickOutside"></div>

这个示例中，指令的参数为 foo，修饰符为 prevent，并绑定了 handleClickOutside 方法作为指令的值。
