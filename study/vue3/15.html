# vue3 组合式api组件事件

在 Vue 3 中使用组合式 API 构建组件时，处理事件的方式与 Vue 2 中稍有不同。组件内部通过使用 emits 选项来声明组件可以触发的事件，然后在组合式 API 中使用 emit 函数来触发这些事件。

以下是使用组合式 API 处理事件的一些常见模式和技巧。

# 声明组件可触发的事件

在 Vue 3 中，我们可以使用 emits 选项来声明组件可以触发的事件。该选项是一个数组，其中的每个字符串表示一个事件的名称。

例如，下面的组件声明了一个名为 selected 的事件：

<template>
  <div>
    <button @click="handleClick">Select</button>
  </div>
</template>

<script>
import { defineEmits } from 'vue';

export default {
  emits: ['selected'],

  setup() {
    function handleClick() {
      // 触发 selected 事件并传递一个字符串参数
      emit('selected', 'hello world');
    }

    return {
      handleClick
    };
  }
}
</script>

在上面的代码中，我们使用了 defineEmits 函数来定义了 emits 选项，然后在组合式 API 的 setup 函数中使用了 emit 函数来触发了 selected 事件，并传递了一个字符串参数 'hello world'。

# 在父组件中监听事件

要在父组件中监听子组件触发的事件，我们需要在子组件上使用 v-on 或 @ 语法来监听该事件。在监听事件时，我们需要使用 v-on:eventName 或 @eventName 语法来指定事件的名称。

例如，下面的代码演示了在父组件中监听名为 selected 的事件：

<template>
  <div>
    <Child @selected="handleSelected" />
  </div>
</template>

<script>
import Child from './Child.vue';

export default {
  components: {
    Child
  },

  methods: {
    handleSelected(message) {
      console.log(message); // 输出 "hello world"
    }
  }
}
</script>

在上面的代码中，我们在 Child 组件上使用了 @selected 语法来监听名为 selected 的事件，并在父组件中定义了 handleSelected 方法来处理该事件。当子组件触发 selected 事件时，父组件中的 handleSelected 方法将会被调用，并且传递子组件触发事件时传递的参数 'hello world'。

# 直接触发父组件中的方法

有时候，在子组件中直接触发父组件中的方法会更加方便。我们可以通过将父组件中的方法作为 prop 传递给子组件，并在子组件中使用 emit 函数触发该方法来实现这一点。

例如，下面的代码演示了如何将 handleSelected 方法作为 prop 传递给子组件，并在子组件中触发该方法：

<template>
  <div>
    <Child :onSelected="handleSelected" />
  </div>
</template>

<script>
import { defineComponent } from 'vue';
import Child from './Child.vue';

export default defineComponent({
  components: {
    Child,
  },
  methods: {
    handleSelected(selectedItem) {
      console.log(`Selected item: ${selectedItem}`);
    },
  },
});
</script>

在子组件中，我们可以使用 emits 选项来声明一个名为 selected 的事件，当用户选择一个项目时，我们可以使用 $emit 方法触发该事件并将选定的项目作为参数传递：

<template>
  <div>
    <ul>
      <li v-for="item in items" :key="item.id" @click="select(item)">
        {{ item.name }}
      </li>
    </ul>
  </div>
</template>

<script>
import { defineEmits, defineProps } from 'vue';

export default {
  emits: ['selected'],
  props: {
    items: {
      type: Array,
      required: true,
    },
  },
  methods: {
    select(item) {
      this.$emit('selected', item);
    },
  },
};
</script>

在父组件中，我们将 handleSelected 方法作为 onSelected prop 传递给子组件，并在子组件中使用 $emit 触发名为 selected 的事件。当子组件触发该事件时，我们可以在 handleSelected 方法中获取选定的项目并进行处理。

除了使用 emits 选项来声明事件，我们还可以使用 provide 和 inject 函数在组合式 API 中传递事件。这种方法允许我们在嵌套层次结构中传递事件，而无需在每个组件中声明事件。

假设我们有一个嵌套层次结构，其中 Parent 组件包含 Child 子组件，而 Child 子组件又包含 Grandchild 孙组件。我们想要在 Grandchild 孙组件中触发一个名为 selected 的事件，并在 Parent 组件中监听该事件并进行处理。

首先，在 Grandchild 孙组件中，我们使用 provide 函数来提供 selected 事件：

<script>
import { provide } from 'vue';
import Child from './Child.vue';

export default {
  setup() {
    const select = (item) => {
      // 提供 selected 事件
      provide('selected', item);
    };

    return {
      select,
    };
  },
};
</script>

然后，在 Parent 组件中，我们使用 inject 函数来获取 selected 事件并在 mounted 生命周期钩子中监听该事件：

<template>
  <div>
    <Child />
  </div>
</template>

<script>
import { defineComponent, inject } from 'vue';
import Child from './Child.vue';

export default defineComponent({
  components: {
    Child,
  },
  mounted() {
    // 获取 selected 事件并监听
    const handleSelected = (selectedItem) => {
      console.log(`Selected item: ${selectedItem}`);
    };
    const selected = inject('selected');
    if (selected) {
      selected.$on('selected', handleSelected);
    }
  },
});
</script>

在 Parent 组件中，我们使用 inject 函数来获取 selected 事件，并在 mounted 生命周期钩子中将其绑定到名为 handleSelected 的事件处理程序函数上。然后，我们使用 $on 方法来监听名为 selected 的事件，并在事件被触发时执行 handleSelected 方法。

最后，在 Child 子组件中，我们可以将 select 方法作为 prop 传递给 Grandchild 孙组件：

<template>
  <div>
    <Grandchild :onSelected="select" />
  </div>
</template>

<script>
import { defineComponent } from 'vue';
import Grandchild from './Grandchild.vue';

export default defineComponent({
  components: {
    Grandchild,
  },
  methods: {
    select(selectedItem) {
      console.log(`Selected item: ${selectedItem}`);
    },
  },
});
</script>

在 Child 子组件中，我们将 select 方法作为 onSelected prop 传递给 Grandchild 孙组件，并在 Grandchild 孙组件中使用 provide 函数提供 selected 事件。当 Grandchild 孙组件触发名为 selected 的事件时，该事件会在整个嵌套层次结构中传递并在 Parent 组件中进行处理。

当子组件中调用 $emit('selected', item) 时，会触发名为 selected 的事件，并将 item 作为参数传递给父组件中绑定的 handleSelected 方法。这样，父组件就可以处理子组件触发的事件，并根据传递的参数进行相应的操作。

另外，使用组合式 API 的方式定义组件时，也可以使用 onXXX 语法糖来注册事件，例如：

import { onMounted, onUnmounted, reactive } from 'vue';

export default {
  setup(props, { emit }) {
    const state = reactive({
      selectedItem: null,
      items: ['item1', 'item2', 'item3']
    });

    const handleSelected = (item) => {
      state.selectedItem = item;
    };

    onMounted(() => {
      console.log('Component mounted');
    });

    onUnmounted(() => {
      console.log('Component unmounted');
    });

    return {
      items: state.items,
      selectedItem: state.selectedItem,
      handleSelected
    };
  }
};

在上面的代码中，我们使用 onXXX 语法糖来注册了组件的 mounted 和 unmounted 事件，同时也定义了一个 handleSelected 方法，用于处理子组件触发的 selected 事件。在 return 语句中，我们将 items、selectedItem 和 handleSelected 作为组件的返回值，供模板中使用。

总结一下，组合式 API 提供了一种更加灵活和直观的方式来定义 Vue 组件。通过 setup 函数中的返回值，我们可以将组件的数据、方法和事件暴露出来，供模板中使用。同时，组合式 API 还提供了一些内置的生命周期钩子和响应式 API，帮助我们更方便地管理组件的状态和行为。
