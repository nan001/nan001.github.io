# vue3 依赖注入

Vue3 组合式 API 提供了一个依赖注入的功能，可以在父组件中注册一个 Provider，将数据传递给所有子组件，子组件可以通过 inject 来注入这些数据。这个功能类似于 React 的 Context。

依赖注入的过程包含两个步骤：

在父组件中注册一个 Provider，将数据传递给所有子组件。 在子组件中使用 inject 来注入这些数据。 下面是一个简单的例子，演示如何在父组件中注册一个 Provider，并在子组件中注入这个 Provider 中的数据。

<template>
  <div>
    <h2>Parent Component</h2>
    <p>Message: {{ message }}</p>
    <ChildComponent />
  </div>
</template>

<script>
import { provide } from 'vue';
import ChildComponent from './ChildComponent.vue';

export default {
  components: {
    ChildComponent,
  },
  setup() {
    provide('message', 'Hello from parent!');
  },
};
</script>

在这个例子中，我们在父组件的 setup() 函数中使用 provide 方法，将一个名为 message 的数据传递给所有子组件。注意，provide 方法只能在 setup() 函数中使用。

接下来，我们在子组件中使用 inject 来注入这个 message 数据。

<template>
  <div>
    <h2>Child Component</h2>
    <p>Message: {{ message }}</p>
  </div>
</template>

<script>
import { inject } from 'vue';

export default {
  setup() {
    const message = inject('message');
    return { message };
  },
};
</script>

在子组件中，我们使用 inject 函数来注入名为 message 的数据。注意，我们需要在子组件的 setup() 函数中使用 inject 函数来获取这个数据。

这样，父组件中的 message 数据就可以传递到子组件中，并且子组件中的 message 数据将随着父组件中的数据的变化而更新。

在 Vue3 中，我们可以使用组合式 API 提供的依赖注入功能来轻松地实现父子组件之间的数据传递，而不必使用 props 或事件。

依赖注入通过 provide 和 inject 方法来实现，父组件可以使用 provide 方法向其所有子组件注入数据，子组件可以使用 inject 方法从父组件中获取数据。

下面是一个示例：

// Parent.vue
import { provide } from 'vue';
import Child from './Child.vue';

export default {
  setup() {
    const message = 'Hello from Parent';
    provide('message', message);
  },
  components: {
    Child,
  },
};

// Child.vue
import { inject } from 'vue';

export default {
  setup() {
    const message = inject('message');
    return {
      message,
    };
  },
};

在上面的示例中，我们在父组件 Parent 中使用 provide 方法向其所有子组件注入一个名为 message 的字符串。在子组件 Child 中，我们使用 inject 方法从父组件中获取 message 数据，并将其赋值给 message 变量。

使用 provide 和 inject 实现依赖注入时，我们需要注意以下几点：

provide 方法只能在 setup 函数中使用，而不能在生命周期函数中使用。 provide 方法的第一个参数是注入数据的 key，可以是任意字符串。但建议使用 Symbol 来作为 key，以避免命名冲突。 inject 方法的第一个参数是要注入的数据的 key，必须与 provide 方法提供的 key 一致。 如果在子组件中使用 inject 方法获取注入的数据，而父组件没有提供该数据，那么 inject 方法将返回 undefined。为了避免这种情况，可以在 inject 方法的第二个参数中指定默认值。 总之，Vue3 的组合式 API 提供了强大的依赖注入功能，可以帮助我们更轻松地实现组件之间的数据传递。

Vue3 组合式 API 中的依赖注入提供了一种在组件树中向下传递数据和功能的方法，而不需要通过 props 或事件来传递。依赖注入在 Vue3 中使用 provide 和 inject 来实现。

在一个组件中，使用 provide 来提供数据或方法，例如：

import { provide } from 'vue';

export default {
  setup() {
    const myData = 'this is my data';
    const myFunction = () => {
      console.log('this is my function');
    };

    provide('myData', myData);
    provide('myFunction', myFunction);

    // ...
  }
}

这个组件提供了两个依赖：一个叫做 myData，另一个叫做 myFunction。这些依赖可以是任何值，包括对象和函数。

在组件树的任何地方，可以使用 inject 来访问提供的依赖：

import { inject } from 'vue';

export default {
  setup() {
    const myData = inject('myData');
    const myFunction = inject('myFunction');

    // 使用 myData 和 myFunction

    // ...
  }
}

在这个示例中，我们使用 inject 方法来注入两个依赖：myData 和 myFunction。这些依赖可以在组件中使用，就像在 provide 中定义的一样。

需要注意的是，使用 inject 来访问提供的依赖时，如果提供的依赖不存在，则会返回 undefined。如果要避免这种情况，可以使用默认值来处理：

const myData = inject('myData', 'default value');

这里，如果 myData 依赖不存在，则会返回 'default value'。

在组件树中使用 provide 和 inject 来传递数据和功能可以减少 props 和事件的使用，从而简化代码。但是，需要小心使用，以避免使组件变得难以理解和维护。通常，只有当在组件树的多个级别上需要共享相同的数据和功能时，才应该使用依赖注入。

组合式 API 中的依赖注入功能，允许我们在组件中注入一些全局的或者局部的依赖，并能够在该组件内部的任何地方访问到这些依赖，而无需显式地通过 props 或者事件传递。

在 Vue3 中，依赖注入的 API 可以通过 provide 和 inject 来实现。

provide 方法用于向下层组件提供依赖。provide 接收两个参数，第一个参数是一个响应式的对象，用于存储提供的依赖；第二个参数是一个可选的 key 值，用于提供对应的 key 值，可以让 inject 的时候更加明确。 inject 方法用于在组件内部注入依赖。inject 接收一个 key 值作为参数，该 key 值应该与 provide 中提供的 key 值相同。inject 方法会从当前组件开始逐级向上查找，直到找到第一个提供该 key 值的父组件，然后将该依赖注入到当前组件中。如果没有找到任何提供该依赖的父组件，则会使用默认值（第二个参数）。 下面是一个示例，演示如何在父组件中使用 provide 方法向下层组件提供依赖，并在子组件中使用 inject 方法获取该依赖：

<template>
  <div>
    <h2>Parent Component</h2>
    <p>Counter: {{ counter }}</p>
    <button @click="increment">Increment</button>
    <ChildComponent />
  </div>
</template>

<script>
import { defineComponent, provide, inject } from 'vue'

const CounterSymbol = Symbol()

export default defineComponent({
  setup() {
    const counter = 0

    const increment = () => {
      counter.value++
    }

    provide(CounterSymbol, counter)

    return {
      counter,
      increment
    }
  }
})

const ChildComponent = defineComponent({
  setup() {
    const counter = inject(CounterSymbol, { value: 0 })

    return {
      counter
    }
  }
})
</script>

在上面的示例中，我们在父组件中使用 provide 方法将 counter 变量向下层组件提供，并使用 CounterSymbol 作为该依赖的 key 值。然后在子组件中使用 inject 方法获取该依赖，并指定 CounterSymbol 作为需要注入的依赖的 key 值。如果没有找到任何提供该依赖的父组件，则会使用默认值 { value: 0 }。

这样，我们就可以在子组件中访问到父组件提供的 counter 变量了。
