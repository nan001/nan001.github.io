# vue3 组合式api组件插槽 Slots

Vue3 组合式 API 组件提供了强大的插槽机制，使得组件的灵活性得到了进一步提升。在组合式 API 组件中，可以使用 slot API 来定义插槽，并使用 v-slot 或 # 语法来在组件的模板中使用插槽。

# 基本用法

定义插槽的方式与传统的 Vue 组件类似，通过 slot API 来定义一个具名插槽：

import { defineComponent } from 'vue';

export default defineComponent({
  name: 'MyComponent',
  setup(props, { slots }) {
    return () => (
      <div>
        <h1>My Component</h1>
        <slot name="header" />
        <slot />
      </div>
    );
  }
});

在上面的示例中，我们定义了两个插槽，一个具名插槽 header，一个匿名插槽。在组件的模板中，我们可以使用 v-slot 或 # 语法来使用这些插槽：

<template>
  <MyComponent>
    <template #header>
      <h2>Header Slot</h2>
    </template>
    <p>Default Slot</p>
  </MyComponent>
</template>

在上面的示例中，我们通过 v-slot 或 # 语法来使用了组件中定义的两个插槽。v-slot 或 # 语法后面可以跟上插槽的名称，如果是匿名插槽，则直接省略名称即可。

# 作用域插槽

除了普通的插槽，Vue3 组合式 API 还支持使用作用域插槽。在 Vue3 中，作用域插槽可以使用 v-slot 或 # 语法来定义，并且可以通过 slotProps 参数将数据传递给插槽。

下面是一个使用作用域插槽的示例：

import { defineComponent } from 'vue';

export default defineComponent({
  name: 'MyComponent',
  setup(props, { slots }) {
    return (props) => (
      <div>
        <h1>{props.title}</h1>
        <slot name="content" slotProps={{ text: 'Hello World' }} />
      </div>
    );
  }
});

在上面的示例中，我们在插槽中使用了 slotProps 参数来将数据传递给插槽。在模板中，我们可以使用 v-slot 或 # 语法来定义作用域插槽：

<template>
  <MyComponent title="My Title">
    <template #content="slotProps">
      <p>{{ slotProps.text }}</p>
    </template>
  </MyComponent>
</template>

在上面的示例中，我们使用了 v-slot 或 # 语法来定义了一个名为 content 的作用插槽，然后在 Child 组件内使用 <slot> 元素将该插槽渲染出来。在父组件中，我们使用 <Child> 组件，并在其中传递了一些内容作为插槽的内容。

除了默认插槽外，Vue3 的组合式 API 还支持具名插槽。具名插槽可以让我们为同一个组件定义多个插槽，并在使用该组件时根据插槽的名称来指定需要使用的插槽。

下面的代码演示了如何在组合式 API 中使用具名插槽：

<!-- Parent.vue -->
<template>
  <Child>
    <template v-slot:header>
      <h2>This is the header</h2>
    </template>
    <template v-slot:footer>
      <p>This is the footer</p>
    </template>
  </Child>
</template>

<!-- Child.vue -->
<template>
  <div>
    <header>
      <slot name="header"></slot>
    </header>
    <main>
      <slot></slot>
    </main>
    <footer>
      <slot name="footer"></slot>
    </footer>
  </div>
</template>

在上面的代码中，我们为 Child 组件定义了两个具名插槽，分别是 header 和 footer。在父组件中，我们使用 v-slot:header 和 v-slot:footer 语法来为这两个插槽指定内容。

在 Child 组件内部，我们使用 <slot> 元素来渲染默认插槽的内容，使用 <slot name="header"> 和 <slot name="footer"> 元素来渲染具名插槽的内容。

通过使用具名插槽，我们可以更灵活地组合和定制组件，提高组件的可复用性和可扩展性。

当我们在组件中使用插槽时，可以通过 setup 函数中的 ctx 参数访问到插槽的内容。具体来说，我们可以通过 ctx.slots 访问到所有插槽的内容，其中插槽的名称就是插槽的属性名。

以下是一个示例组件，它包含一个具名插槽和一个默认插槽：

<template>
  <div>
    <h2>{{ title }}</h2>
    <div class="content">
      <slot name="content">
        <p>Default content</p>
      </slot>
    </div>
    <div class="footer">
      <slot></slot>
    </div>
  </div>
</template>

<script>
import { defineComponent } from 'vue';

export default defineComponent({
  props: {
    title: {
      type: String,
      required: true
    }
  },
  setup(props, ctx) {
    return {
      slots: ctx.slots
    };
  }
});
</script>

在这个组件中，我们定义了一个 title 属性，并且在模板中使用了一个具名插槽 content 和一个默认插槽。在 setup 函数中，我们通过 ctx.slots 访问到了插槽的内容，并将其存储在了 slots 变量中。

现在，我们可以在父组件中使用这个组件，并为它的插槽提供内容。下面是一个示例：

<template>
  <div>
    <MyComponent title="My Title">
      <template v-slot:content>
        <p>This is the content of the named slot.</p>
      </template>
      <p>This is the content of the default slot.</p>
    </MyComponent>
  </div>
</template>

<script>
import { defineComponent } from 'vue';
import MyComponent from './MyComponent.vue';

export default defineComponent({
  components: {
    MyComponent
  }
});
</script>

在这个示例中，我们使用了 <template v-slot:content> 语法来为具名插槽 content 提供了一个 <p> 元素作为内容，同时也为默认插槽提供了一个 <p> 元素。

当 MyComponent 组件渲染时，它会将父组件中提供的插槽内容插入到组件中相应的插槽位置上。在 setup 函数中，我们可以通过 ctx.slots 访问到插槽的内容，并根据需要进行处理。

在组合式 API 中，我们可以使用 createSlots 方法来创建插槽，并将它们传递给子组件。

createSlots 方法接受一个对象作为参数，该对象的键表示插槽名称，值表示要分配到插槽的组件模板。例如，我们可以在父组件中这样使用：

<template>
  <Child>
    <!-- 分配名为 header 的插槽 -->
    <template #header>
      <h1>{{ title }}</h1>
    </template>
    
    <!-- 分配默认插槽 -->
    <template>
      <p>{{ content }}</p>
    </template>
  </Child>
</template>

<script>
import { defineComponent, createSlots } from 'vue'
import Child from './Child.vue'

export default defineComponent({
  components: {
    Child
  },
  data() {
    return {
      title: 'Hello World',
      content: 'This is some content'
    }
  },
  setup() {
    // 创建插槽对象
    const slots = createSlots({
      header: () => [<h2>Hello from parent</h2>]
    })

    return {
      slots
    }
  }
})
</script>

在子组件中，我们可以使用 v-slot 或 # 语法来定义插槽的位置：

<template>
  <div>
    <!-- 使用名为 header 的插槽 -->
    <slot name="header"></slot>

    <!-- 使用默认插槽 -->
    <slot></slot>
  </div>
</template>

注意，createSlots 方法返回的对象中的函数是一个返回一个数组的函数。这是因为 Vue 3 中支持多个内容的插槽，因此我们需要将要分配到插槽的多个组件模板包装在一个数组中。

此外，我们还可以通过 createSlots 方法传递 props 给插槽模板。例如，我们可以修改上面的示例，在 createSlots 方法中传递一个包含 headerText 属性的对象，并在插槽模板中使用它：

<template>
  <Child>
    <!-- 分配名为 header 的插槽 -->
    <template #header="props">
      <h1>{{ props.headerText }}</h1>
    </template>
    
    <!-- 分配默认插槽 -->
    <template>
      <p>{{ content }}</p>
    </template>
  </Child>
</template>

<script>
import { defineComponent, createSlots } from 'vue'
import Child from './Child.vue'

export default defineComponent({
  components: {
    Child
  },
  data() {
    return {
      headerText: 'Hello World',
      content: 'This is some content'
    }
  },
  setup() {
    // 创建带有 props 的插槽对象
    const slots = createSlots({
      header: ({ headerText }) => [<h2>{headerText}</h2>]
    }, { headerText: this.headerText })

    return {
      slots
    }
  }
})
</script>

在这个示例中，我们将 headerText 属性传递给 createSlots 方法，并在插槽模板中使用headerText 属性渲染了一个标题。这种方式可以让我们在组件中使用插槽的同时，也能动态地传递一些属性值给插槽模板，从而增加了组件的灵活性和可复用性。

另外，需要注意的是，在组合式 API 组件中使用插槽时，我们需要通过 slotProps 来接收传递给插槽的 props。在上面的示例中，我们在 createSlots 函数中定义了 slotProps 参数来接收传递给插槽的 props，并在插槽模板中使用了这个参数来访问这些 props。这样做的目的是为了更好地封装组件的内部实现，同时也可以避免父组件直接操作插槽的 props，提高组件的健壮性和可维护性。
