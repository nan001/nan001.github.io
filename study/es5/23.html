# Object

# Object.assign() (ES6 新增)

Object.assign() 是 ES6 新增的静态方法，用于将一个或多个源对象的所有可枚举属性复制到目标对象中，并返回目标对象。它采用的是浅拷贝的方式，即如果属性的值是一个对象，则复制的是这个对象的引用。

# 语法：

Object.assign(target, ...sources)

# 参数：

target：目标对象。
sources：一个或多个源对象。

# 返回值：

返回目标对象。

# 下面是一个简单的示例，演示如何使用 Object.assign() 方法将多个对象合并为一个对象：

const obj1 = { a: 1, b: 2 };
const obj2 = { c: 3, d: 4 };
const obj3 = { e: 5, f: 6 };
const mergedObj = Object.assign({}, obj1, obj2, obj3);
console.log(mergedObj); // { a: 1, b: 2, c: 3, d: 4, e: 5, f: 6 }

在这个示例中，我们定义了三个对象 obj1、obj2、obj3，并使用 Object.assign() 方法将它们合并为一个新的对象 mergedObj。注意，我们传递了一个空对象 {} 作为第一个参数，这样就可以保证 Object.assign() 方法不会改变任何已有的对象。

# 下面再来看一个复杂一点的示例，演示如何使用 Object.assign() 方法进行对象的深拷贝：

function deepCopy(obj) {
  if (typeof obj !== 'object' || obj === null) {
    return obj;
  }
  
  const newObj = {};
  for (const key in obj) {
    if (Object.hasOwnProperty.call(obj, key)) {
      newObj[key] = deepCopy(obj[key]);
    }
  }
  return newObj;
}

const obj1 = {
  a: 1,
  b: {
    c: 2,
    d: {
      e: 3,
      f: 4
    }
  }
};

const obj2 = Object.assign({}, obj1);
obj2.b.c = 5;
obj2.b.d.e = 6;
console.log(obj1); // { a: 1, b: { c: 2, d: { e: 6, f: 4 } } }
console.log(obj2); // { a: 1, b: { c: 5, d: { e: 6, f: 4 } } }

const obj3 = deepCopy(obj1);
obj3.b.c = 7;
obj3.b.d.e = 8;
console.log(obj1); // { a: 1, b: { c: 2, d: { e: 6, f: 4 } } }
console.log(obj3); // { a: 1, b: { c: 7, d: { e: 8, f: 4 } } }

在这个示例中，我们定义了一个名为 deepCopy() 的函数，用于对对象进行深拷贝。然后我们定义了一个包含嵌套对象的原始对象 obj1，并使用 Object.assign() 方法将它复制到 obj2 中。然后我们改变 obj2 中的嵌套对象的值，并打印出 obj1 和 obj2 的值，以便比较它们之间的差异。

function deepCopy(obj) {
  if (obj === null || typeof obj !== "object") {
    return obj;
  }

  const copy = Object.assign(Array.isArray(obj) ? [] : {}, obj);

  Object.keys(copy).forEach(key => {
    copy[key] = deepCopy(copy[key]);
  });

  return copy;
}

const obj1 = {
  a: 1,
  b: {
    c: 2,
    d: 3
  }
};

const obj2 = Object.assign({}, obj1);

obj2.b.c = 4;

console.log(obj1); // { a: 1, b: { c: 2, d: 3 } }
console.log(obj2); // { a: 1, b: { c: 4, d: 3 } }

需要注意的是，Object.assign() 方法只是将源对象中的可枚举属性复制到目标对象中。如果源对象中有一个不可枚举属性，它将不会被复制。此外，如果在复制过程中出现重复的属性，目标对象中的属性值将被覆盖为源对象中的属性值。如果您想要进行深拷贝，请使用类似上面示例中的 deepCopy() 函数。

# Object.create()

Object.create() 是 ES5 中一个创建新对象的方法。它的作用是使用指定的原型对象创建一个新对象。新对象的原型就是我们传入的原型对象。下面是 Object.create() 方法的详解和示例。

# 语法

Object.create(proto, [propertiesObject])

# 参数

proto: 新创建对象的原型对象。
propertiesObject (可选): 用来描述新对象的属性的对象。该参数对象的属性包含其可枚举属性的名称和属性描述符。属性描述符可以是数据描述符，也可以是存取描述符。

# 返回值

一个新对象，它的原型为 proto。

# 注意事项

proto 参数必须是一个对象或者 null。如果不是对象类型，会抛出一个 TypeError 错误。
propertiesObject 参数可以包含属性描述符对象，用于指定新对象的属性。属性描述符对象的格式与 Object.defineProperties() 方法的参数一致。
如果 propertiesObject 参数不是 null，则会创建一个新的对象，并将 propertiesObject 中的属性添加到新对象中。如果 propertiesObject 参数为 null 或未提供，则返回一个没有任何属性和方法的空对象。

# 示例

# 创建空对象

可以使用 Object.create() 方法来创建一个空对象。空对象没有任何属性和方法。

const emptyObj = Object.create(null);

console.log(emptyObj); // {}
# 创建带原型对象的对象
// 定义一个原型对象
const person = {
  fullName() {
    return `${this.firstName} ${this.lastName}`;
  }
};

// 创建一个新对象
const johnDoe = Object.create(person);

// 设置新对象的属性
johnDoe.firstName = 'John';
johnDoe.lastName = 'Doe';

console.log(johnDoe.fullName()); // "John Doe"

在这个示例中，我们创建了一个 person 对象作为原型。然后，我们使用 Object.create() 方法创建了一个新对象 johnDoe，并将 person 对象作为 johnDoe 对象的原型。最后，我们为 johnDoe 对象添加了 firstName 和 lastName 属性，并使用 johnDoe.fullName() 方法来调用 person 对象中定义的 fullName() 方法。

# 创建带属性的对象
const obj = Object.create({}, {
  name: {
    value: 'John',
    enumerable: true
  },
  age: {
    value: 30,
    enumerable: true
  }
});

console.log(obj); // { name: 'John', age: 30 }

在这个示例中，我们使用 Object.create() 方法创建了一个新对象，并为该对象添加了 name 和 age 属性。propertiesObject 参数对象包含了属性描述符，用于指定新对象的属性。value 属性指定了属性的值，enumerable 属性指定了属性是否可枚举。

# 继承 Array 对象
// 创建新对象
const arr = Object.create(Array.prototype);

// 设置新对象的元素
arr.push(1).push(2).push(3);

console.log(arr); // 输出 [1, 2, 3]

在这个示例中，我们使用 Object.create() 方法创建了一个新对象 arr，并将它的原型设置为 Array.prototype，这样它就可以继承 Array 对象的方法和属性。然后我们可以使用 push() 方法向 arr 中添加元素，最后输出 arr 的值为 [1, 2, 3]。

# Object.defineProperty()

Object.defineProperty() 方法用于在一个对象上定义一个新属性，或者修改一个对象的现有属性，并返回这个对象。它允许我们定义一个新属性，或者修改一个已有属性的行为，例如改变它的 writable、enumerable、configurable 属性。

# 语法如下：

Object.defineProperty(obj, prop, descriptor)

# 参数说明：

obj：必须，需要定义属性的对象。
prop：必须，定义或修改的属性的名称。
descriptor：必须，定义或修改的属性的描述符对象。

# 属性描述符（descriptor）是一个普通的 JavaScript 对象，它有以下可选键值：

configurable：默认为 false，当且仅当该属性的 configurable 键值为 true 时，该属性的描述符才能够被改变，同时该属性也能从对应的对象上被删除。
enumerable：默认为 false，当且仅当该属性的 enumerable 键值为 true 时，该属性才会出现在对象的枚举属性中。
value：该属性对应的值。可以是任何有效的 JavaScript 值（数值，对象，函数等）。
writable：默认为 false，当且仅当该属性的 writable 键值为 true 时，属性的值才能被赋值运算符改变。
get：给属性提供 getter 函数，当访问该属性时，会调用此函数。执行时不传入任何参数，但是会传入 this 对象（由于继承关系，这里的 this 并不一定是定义该属性的对象）。该函数的返回值会被用作属性的值。
set：给属性提供 setter 函数，当属性值被修改时，会调用此函数。该方法接受一个参数（也就是被赋予的新值），会将新值存储在对应的属性上。

# 下面是一个使用 Object.defineProperty() 定义属性的例子：

const obj = {};
Object.defineProperty(obj, 'name', {
  value: 'Jack',
  writable: false,
  configurable: false,
  enumerable: true
});
console.log(obj.name); // Jack
obj.name = 'Tom'; // Cannot assign to read only property 'name' of object '#<Object>'
console.log(obj.name); // Jack
delete obj.name; // Cannot delete property 'name' of object '#<Object>'
console.log(obj); // { name: 'Jack' }

在上面的代码中，我们定义了一个名为 name 的属性，并设置了它的值为 'Jack'。同时，我们将属性的 writable 键值设为 false，表示该属性不可被修改。由于 configurable 也被设置为 false，因此无法删除该属性。最后，我们将 enumerable 设为 true，表示该属性会出现在对象的枚举属性中。

# 除此之外，我们还可以使用 Object.defineProperty() 定义 getter 和 setter，getter 方法用于获取属性值，setter 方法用于设置属性值。如下所示：

const obj = {};

Object.defineProperty(obj, 'fullName', {
  get: function() {
    return `${this.firstName} ${this.lastName}`;
  },
  set: function(name) {
    const parts = name.split(' ');
    this.firstName = parts[0];
    this.lastName = parts[1];
  }
});

obj.firstName = 'John';
obj.lastName = 'Doe';

console.log(obj.fullName); // "John Doe"

obj.fullName = 'Jane Smith';

console.log(obj.firstName); // "Jane"
console.log(obj.lastName); // "Smith"

在上面的示例中，我们定义了一个空对象 obj，并使用 Object.defineProperty() 定义了一个名为 fullName 的属性，其中包含了一个 getter 和一个 setter 方法。在 getter 方法中，我们将 firstName 和 lastName 属性合并为完整的姓名。在 setter 方法中，我们将传入的姓名字符串拆分成 firstName 和 lastName，并设置相应的属性值。

然后我们设置了 obj 的 firstName 和 lastName 属性，并通过 console.log() 打印了 fullName 属性的值。最后，我们将 fullName 属性设置为了一个新的字符串，这时 setter 方法被调用，相应的 firstName 和 lastName 属性也被设置为了新值，最后再次打印输出 firstName 和 lastName 的值。

# Object.defineProperties()

Object.defineProperties() 方法可以一次性定义一个对象的多个属性。它接受两个参数：要定义属性的对象和一个属性描述符对象。

# 一个属性描述符对象可以包含以下属性：

configurable：如果为 true，则该属性可以使用 delete 删除，可以通过属性描述符进行修改，可以转换为数据属性。默认值为 false。
enumerable：如果为 true，则该属性可以在 for...in 循环中枚举。默认值为 false。
value：该属性的值。不能与 get 或 set 同时存在。
writable：如果为 true，则该属性的值可以被修改。默认值为 false。
get：读取该属性时调用的函数。不能与 value 同时存在。
set：写入该属性时调用的函数。不能与 value 同时存在。

# 下面是一个示例：

const obj = {};
Object.defineProperties(obj, {
  property1: {
    value: 42,
    writable: false
  },
  property2: {
    value: 'hello',
    writable: true
  }
});

在上面的示例中，我们定义了一个名为 obj 的空对象，并使用 Object.defineProperties() 方法定义了两个属性：property1 和 property2。

property1 的值为 42，并且是只读的，因为我们将 writable 属性设置为 false。property2 的值为 'hello'，并且是可写的，因为我们将 writable 属性设置为 true。

使用 Object.defineProperties() 方法定义多个属性时，属性描述符对象可以使用同一个格式。

# Object.entries() ES8（ES2017）新增

Object.entries() 是 ES8（ES2017）中引入的一个方法，它返回一个给定对象自身可枚举属性的键值对数组，数组的每一项都是一个形如 [key, value] 的数组。

# 下面是 Object.entries() 的语法：

Object.entries(obj)

其中，obj 是要返回其属性的对象。

# 下面是一个简单的示例，展示如何使用 Object.entries() 方法遍历对象的键值对：

const person = {
  firstName: 'John',
  lastName: 'Doe',
  age: 30
};

for (const [key, value] of Object.entries(person)) {
  console.log(`${key}: ${value}`);
}
// Output:
// firstName: John
// lastName: Doe
// age: 30

在上面的示例中，我们定义了一个名为 person 的对象，包含了三个属性：firstName、lastName 和 age。然后我们使用 Object.entries() 方法将该对象的键值对作为数组返回，并使用 for...of 循环遍历这个数组，输出每一个键值对的键和值。

需要注意的是，Object.entries() 方法只会返回对象自身的可枚举属性的键值对。如果要遍历对象的所有属性，可以使用 for...in 循环或者 Object.getOwnPropertyNames() 方法。同时，Object.entries() 方法的返回值是一个数组，而不是一个对象，因此不能再使用对象的方法和属性。

# Object.freeze()

Object.freeze() 方法用于冻结对象，防止对已有属性的修改，添加或删除。它返回被冻结的对象。被冻结的对象无法被修改、添加或删除属性，但不会阻止属性值为引用类型的属性进行修改。

下面是一个使用 Object.freeze() 方法的示例：

const obj = {
  name: 'John',
  age: 30
};

Object.freeze(obj);  // 冻结对象

obj.name = 'Jane';  // 尝试修改属性
obj.gender = 'male';  // 尝试添加属性

console.log(obj);  // { name: 'John', age: 30 }

在上面的代码中，我们使用 Object.freeze() 方法冻结了 obj 对象。然后我们尝试修改 name 属性的值和添加一个新的 gender 属性，但由于 obj 对象已被冻结，所以这些操作都不会成功。最后，我们将 obj 对象打印到控制台上，可以看到它的属性值没有发生变化。

需要注意的是，Object.freeze() 方法只能冻结对象的第一层属性，如果对象的属性是引用类型，那么这些引用类型的属性不会被冻结。如果需要冻结对象的所有属性，可以使用深度冻结方法。

# Object.fromEntries() ES2019 新增

Object.fromEntries() 方法是 ES2019 引入的新方法，它可以从键值对数组中创建一个新对象。它的语法如下：

Object.fromEntries(iterable)

其中，iterable 是一个键值对数组或者其他可迭代的对象，例如 Map 对象。

Object.fromEntries() 方法将迭代 iterable 对象中的每一个键值对，并将它们添加到一个新的对象中。每一个键值对都应该是一个数组，第一个元素是键名，第二个元素是键值。

下面是一个使用 Object.fromEntries() 方法创建新对象的例子：

const arr = [['a', 1], ['b', 2], ['c', 3]];
const obj = Object.fromEntries(arr);
console.log(obj); // {a: 1, b: 2, c: 3}

在这个例子中，我们定义了一个包含三个键值对的数组 arr，然后使用 Object.fromEntries() 方法将其转换成一个新的对象 obj。obj 的内容与 arr 的内容相同，但是它是一个对象，而不是一个数组。

除了数组以外，Object.fromEntries() 方法还可以接受其他可迭代的对象作为参数，例如 Map 对象。下面是一个使用 Map 对象创建新对象的例子：

const map = new Map([['a', 1], ['b', 2], ['c', 3]]);
const obj = Object.fromEntries(map);
console.log(obj); // {a: 1, b: 2, c: 3}

在这个例子中，我们定义了一个 Map 对象 map，然后使用 Object.fromEntries() 方法将其转换成一个新的对象 obj。

需要注意的是，Object.fromEntries() 方法是 ES2019 引入的新方法，在一些较老的浏览器中可能不支持。如果需要在较老的浏览器中使用该方法，可以使用 polyfill 库进行兼容。

# Object.getOwnPropertyDescriptor()

Object.getOwnPropertyDescriptor() 方法返回指定对象上一个自有属性对应的属性描述符（自有属性是指直接赋予该对象的属性，不需要从原型链上进行查找）。

# 属性描述符是一个对象，包含以下属性：

configurable：表示是否可以通过 delete 删除属性或修改属性特性，默认值为 false。
enumerable：表示是否可以通过 for...in 循环遍历到属性，默认值为 false。
value：表示属性值。
writable：表示属性是否可以被赋值运算符改变其值，默认值为 false。
get：表示获取属性值的方法。
set：表示设置属性值的方法。

# 语法：

Object.getOwnPropertyDescriptor(obj, prop)

# 参数：

obj：要查找的对象。
prop：要查找的属性名。

# 返回值：

一个对象，描述了所请求的属性的属性描述符。

# 示例：

const obj = {
  name: 'Alice',
  age: 18,
  get fullName() {
    return this.name;
  },
  set fullName(value) {
    this.name = value;
  }
};

const descriptor = Object.getOwnPropertyDescriptor(obj, 'age');
console.log(descriptor);
// 输出：{ value: 18, writable: true, enumerable: true, configurable: true }

const fullNameDescriptor = Object.getOwnPropertyDescriptor(obj, 'fullName');
console.log(fullNameDescriptor);
// 输出：{ get: [Function: get fullName], set: [Function: set fullName], enumerable: true, configurable: true }

在上面的示例中，我们首先定义了一个包含普通属性和访问器属性的对象 obj。然后，我们使用 Object.getOwnPropertyDescriptor() 方法获取了 obj 上的 age 属性和 fullName 属性的属性描述符，并将其打印出来。

可以看到，age 属性的属性描述符包含了 value、writable、enumerable 和 configurable 四个属性，而 fullName 属性的属性描述符包含了 get、set、enumerable 和 configurable 四个属性。

# Object.getOwnPropertyDescriptors()

Object.getOwnPropertyDescriptors() 方法返回指定对象的所有自身属性的描述符。如果对象中没有属性，则返回空对象。

# 语法：

Object.getOwnPropertyDescriptors(obj)

# 参数：

obj：要返回属性描述符的对象。

# 返回值：

一个对象，其中包含指定对象的所有自身属性的属性描述符。

# 示例：

const obj = {
  foo: 'bar',
  get baz() {
    return this.foo;
  }
};

const descriptors = Object.getOwnPropertyDescriptors(obj);

console.log(descriptors.foo);
// { value: 'bar', writable: true, enumerable: true, configurable: true }

console.log(descriptors.baz);
// { get: [Function: get baz], set: undefined, enumerable: true, configurable: true }

在上面的示例中，我们定义了一个包含两个属性的对象 obj，其中 baz 属性是一个 getter 函数。我们使用 Object.getOwnPropertyDescriptors() 方法获取对象 obj 的所有属性描述符，并将它们存储在 descriptors 变量中。最后，我们使用 console.log() 方法输出了 foo 属性和 baz 属性的描述符。

# Object.is() ES6 新增

Object.is() 是 ES6 引入的方法，用于比较两个值是否相同，返回一个布尔值。

# 与 === 的区别在于：

Object.is() 认为 NaN 与自身相等，而 === 不认为。
Object.is() 认为 0 和 -0 不相等，而 === 认为相等。

# 语法如下：

Object.is(value1, value2)

# 参数说明：

value1：第一个要比较的值。
value2：第二个要比较的值。

# 返回值：

如果 value1 与 value2 相等，则返回 true，否则返回 false。

# 示例：

console.log(Object.is(2, 2)); // true
console.log(Object.is(0, 0)); // true
console.log(Object.is(-0, -0)); // true
console.log(Object.is(0, -0)); // false
console.log(Object.is(NaN, NaN)); // true
console.log(Object.is({}, {})); // false
console.log(Object.is(null, null)); // true
console.log(Object.is(undefined, null)); // false

在上面的示例中，我们使用 Object.is() 比较了一些不同类型的值。可以看出，只有当值相等时，Object.is() 才会返回 true。

# Object.isExtensible()

Object.isExtensible() 方法是用于判断一个对象是否可扩展的。如果一个对象可扩展，则可以向其添加新属性。该方法返回一个布尔值，如果对象可扩展则返回 true，否则返回 false。

# 语法：

Object.isExtensible(obj)

# 参数：

obj: 要检查的对象。

# 示例：

console.log(Object.isExtensible(obj1)); // false

const obj2 = { b: 2 };
console.log(Object.isExtensible(obj2)); // true

Object.freeze(obj2); // 冻结对象
console.log(Object.isExtensible(obj2)); // false

在上面的示例中，首先创建了两个对象 obj1 和 obj2。调用 Object.isExtensible() 方法检查这两个对象是否可扩展。对于 obj1，因为没有使用任何方法来改变其可扩展性，因此返回 true。对于 obj2，使用 Object.freeze() 冻结了对象，因此返回 false。

# Object.isFrozen()

Object.isFrozen() 方法是一个用于判断对象是否被冻结（不可扩展、不可写、不可配置）的方法。如果一个对象被冻结，则该方法返回 true，否则返回 false。

Object.isFrozen() 方法接受一个参数，即需要检测的对象。

# 示例：

const obj1 = {
  prop1: 'value1'
};

// 将 obj1 冻结
Object.freeze(obj1);

console.log(Object.isFrozen(obj1)); // true

// 尝试删除 prop1 属性
delete obj1.prop1; // 不会起作用，因为对象被冻结了

// 尝试修改 prop1 属性
obj1.prop1 = 'new value'; // 不会起作用，因为对象被冻结了

// 尝试添加新属性
obj1.prop2 = 'value2'; // 不会起作用，因为对象被冻结了

console.log(obj1); // { prop1: 'value1' }

在上面的示例中，我们使用 Object.freeze() 方法将 obj1 冻结，并使用 Object.isFrozen() 方法检测 obj1 是否被冻结。然后我们尝试删除 prop1 属性、修改 prop1 属性以及添加新属性 prop2，但都不会起作用，因为对象被冻结了。最后输出 obj1，发现它仍然是 { prop1: 'value1' }，也就是说对象没有被修改。

需要注意的是，Object.isFrozen() 方法只能检测对象本身是否被冻结，无法检测对象的属性是否被冻结。如果需要检测对象属性的冻结状态，可以使用 Object.getOwnPropertyDescriptor() 方法获取属性的描述符，并检查 writable 和 configurable 属性是否为 false。

# 参数：

# 返回值：

# 示例：

  prop2: 2
};

// 将 obj 封闭

在上面的示例中，我们创建了一个包含两个属性的对象 obj，然后使用 Object.seal() 将其封闭。最后，我们使用 Object.isSealed() 检查对象是否被封闭，并输出结果为 true。

# Object.preventExtensions()

Object.preventExtensions() 是一个内置的对象方法，用于防止对象被扩展。这意味着，一旦对象被标记为不可扩展，就无法添加新的属性。但是，现有的属性可以更改或删除。

在上面的示例中，我们首先创建了一个名为 obj 的对象，并给它添加了两个属性 name 和 age。然后，我们使用 Object.preventExtensions() 方法防止了 obj 对象的扩展。最后，我们试图添加一个名为 gender 的新属性，但它没有产生任何效果。

# Object.prototype.hasOwnProperty()

# Object.prototype.isPrototypeOf()

Object.prototype.isPrototypeOf() 方法用于测试一个对象是否存在于另一个对象的原型链上。该方法返回一个布尔值，表示被检查的对象是否为调用该方法的对象的原型。

# 语法：

prototypeObj.isPrototypeOf(object)

# 参数：

prototypeObj：要检查的原型对象。
object：要检查的对象。

# 返回值：

如果 prototypeObj 是 object 的原型或者位于 object 的原型链上，则返回 true。
否则返回 false。

# 示例：

  firstName: 'John',
  lastName: 'Doe',
  age: 25,
  fullName: function() {
    return this.firstName + ' ' + this.lastName;
  }
};

// 创建一个对象，该对象的原型为 person
const employee = Object.create(person);
employee.empCode = 'E001';

// 测试 person 是否是 employee 的原型
console.log(person.isPrototypeOf(employee)); // true

// 创建一个对象，该对象的原型为 employee
const manager = Object.create(employee);
manager.department = 'Sales';

// 测试 person 是否是 manager 的原型
console.log(person.isPrototypeOf(manager)); // true

// 测试 employee 是否是 manager 的原型
console.log(employee.isPrototypeOf(manager)); // true

在上面的例子中，我们定义了一个原型对象 person，并使用 Object.create() 方法创建了两个新对象 employee 和 manager，它们的原型分别设置为 person 和 employee。

然后，我们使用 isPrototypeOf() 方法来检查 person 是否是 employee 和 manager 的原型。因为 employee 的原型是 person，而 manager 的原型是 employee，所以 person 是它们的共同原型，所以这些检查都返回 true。

Object.prototype.toString() 是一个 JavaScript 内置方法，用于将一个对象转换为一个字符串表示形式。它常常用于调试和输出对象信息，其返回值为一个字符串。

在JavaScript中，所有对象都有一个 constructor 属性，该属性指向用于创建该对象的构造函数。因此，使用对象的 constructor 属性可以检查它的类型或是否是特定构造函数的实例。

# obj.hasOwnProperty()

obj.hasOwnProperty() 是一个 JavaScript 对象的方法，它返回一个布尔值，指示对象自身属性中是否具有指定的属性。如果对象有指定属性，返回 true，否则返回 false。

# obj.isPrototypeOf()

obj.isPrototypeOf() 方法是用于检查一个对象是否存在于另一个对象的原型链上，返回布尔值。具体来说，它用于判断一个对象是否是另一个对象的原型。

在上面的例子中，我们定义了一个原型对象 protoObj，并创建了一个新对象 obj，该对象使用 Object.create() 方法将其原型指向 protoObj。然后我们分别使用 isPrototypeOf() 方法检查 obj 是否是 protoObj 的实例，以及 protoObj 是否是 obj 的实例。第一次检查返回 true，因为 obj 是 protoObj 的实例，第二次检查返回 false，因为 protoObj 不是 obj 的实例。

// 判断 toString 属性是否可枚举
console.log(obj.propertyIsEnumerable('toString')); // false

在上面的示例中，obj.propertyIsEnumerable('name') 返回 true，因为 name 属性是 obj 对象自己的属性，并且可枚举。而 obj.propertyIsEnumerable('toString') 返回 false，因为 toString 属性是 Object.prototype 的属性，不是 obj 对象自己的属性，且默认情况下不可枚举。

# obj.toLocaleString()

obj.toLocaleString() 方法返回一个字符串，表示该对象的本地化字符串。

该方法可以被所有对象继承。当调用该方法时，JavaScript 引擎会首先查看该对象是否有 toLocaleString() 方法，如果有，则会调用该方法并返回其结果；如果没有，则会调用该对象的 toString() 方法并返回其结果。

该方法通常由本地化的子类实现，以返回适当的本地化字符串。例如，当你调用 toLocaleString() 方法时，日期对象会返回一个格式化的日期字符串，数字对象会返回一个格式化的数字字符串，而数组对象会返回一个包含元素本地化字符串的数组字符串。

下面是一个 toLocaleString() 方法的示例：

const date = new Date(Date.UTC(2023, 3, 29, 8, 30, 0));
const str = date.toLocaleString('en-US', { timeZone: 'America/New_York' });
console.log(str); // "4/29/2023, 4:30:00 AM"

在这个示例中，我们创建了一个日期对象 date，然后使用 toLocaleString() 方法将其转换为本地化的字符串，并指定了 en-US 作为区域设置，以及 America/New_York 作为时区。最后，我们将结果打印到控制台中，输出为 "4/29/2023, 4:30:00 AM"。

# obj.toString()

toString() 是 Object 的原型方法，它用于将对象转换成字符串。

# 语法：

obj.toString()

# 示例：

const obj = {a: 1, b: 2};
console.log(obj.toString()); // [object Object]

注意：toString() 返回的字符串一般都是包含 [object] 的字符串，其中 object 表示对象类型，如 [object Object]。如果要对对象进行自定义字符串转换，可以重写该方法。

# obj.valueOf()

valueOf() 方法是 JavaScript 中的一个内置方法，用于返回当前对象的原始值，通常作为类型转换时的操作。该方法通常会被隐式调用，例如当对象需要被转换为字符串时。如果需要明确地调用该方法，可以使用 Object.prototype.valueOf() 或在自定义对象中覆盖该方法。

# 语法：

obj.valueOf()

# 示例：

const obj = {
  x: 10,
  y: 20,
  valueOf: function() {
    return this.x + this.y;
  }
};

console.log(obj.valueOf()); // 输出 30
console.log(Number(obj)); // 输出 30

在上面的示例中，我们定义了一个包含 x 和 y 属性的对象 obj，并覆盖了它的 valueOf() 方法，返回 x 和 y 的和。我们可以通过调用 obj.valueOf() 显式地调用该方法，并通过将 obj 转换为数字来隐式调用该方法，并将其作为数字返回。
