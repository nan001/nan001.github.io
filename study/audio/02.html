# audio HTML5音频标签<audio>支持的属性和事件：

# 属性

属性 描述
src 音频文件的URL
preload 预加载音频文件的方式。可选值：none、metadata、auto
autoplay 自动播放音频。
loop 循环播放音频。
muted 静音音频。
controls 显示音频控件。
volume 设置音量。可选值：0.0（静音）到1.0（最大音量）。
currentTime 设置音频的当前播放时间，单位为秒。
duration 音频的总时长，单位为秒。

# abort 音频加载被中止。

abort是HTML5 <audio>标签的一个事件，当用户手动停止音频的加载或音频文件下载被终止时，就会触发abort事件。

下面是一个简单的示例，演示了如何监听abort事件：

<audio src="audio.mp3" controls></audio>

<script>
const audio = document.querySelector('audio');
audio.addEventListener('abort', () => {
  console.log('Audio loading was aborted');
});
</script>

在这个例子中，我们将一个名为“audio.mp3”的音频文件作为<audio>标签的源文件，然后使用addEventListener()方法监听abort事件。当音频文件的加载被中断时，控制台将打印“Audio loading was aborted”消息。

abort事件还可以用于手动中止音频的加载。例如，您可以在用户按下“停止”按钮时调用audioElement.abort()方法来中止音频文件的下载。

需要注意的是，abort事件只在音频文件的下载过程中被触发，而不是在播放过程中。如果您需要在音频播放中捕获错误，请考虑使用其他事件，如error或ended事件。

# canplay 音频文件已经可以开始播放。

canplay是HTML5 <audio>标签的一个事件，当浏览器可以开始播放音频时（至少可以播放一部分）就会触发该事件。

下面是一个简单的示例，演示了如何监听canplay事件：

<audio src="audio.mp3" controls></audio>

<script>
const audio = document.querySelector('audio');
audio.addEventListener('canplay', () => {
  console.log('Audio can start playing');
});
</script>

在这个例子中，我们将一个名为“audio.mp3”的音频文件作为<audio>标签的源文件，然后使用addEventListener()方法监听canplay事件。当浏览器准备好开始播放音频时，控制台将打印“Audio can start playing”消息。

需要注意的是，canplay事件不保证音频可以完全播放，因为在播放过程中可能会发生其他错误。如果您需要检查音频是否完全加载，请考虑使用其他事件，如canplaythrough事件。

此外，还有许多其他的<audio>事件可以用于监视和控制音频的播放，例如play、pause、ended等事件。

此外，还有许多其他的<audio>事件可以用于监视和控制音频的播放，例如play、pause、ended等事件。

需要注意的是，durationchange事件可能会在音频加载后立即触发，但也可能需要一些时间才能触发。这取决于浏览器已经加载的音频数据量以及音频本身的大小。因此，如果您需要获取音频的总时长，请确保在durationchange事件触发后再获取它。

此外，还有许多其他的<audio>事件可以用于监视和控制音频的播放，例如play、pause、ended等事件。

此外，还有许多其他的<audio>事件可以用于监视和控制音频的播放，例如play、pause、ended等事件。

此外，还有许多其他的<audio>事件可以用于监视和控制音频的播放，例如play、pause、timeupdate等事件。

# progress 音频文件正在下载。

progress是HTML5 <audio>标签的一个事件，它在浏览器下载音频文件时定期触发，以便向用户提供下载进度信息。

下面是一个简单的示例，演示了如何监听progress事件：

<audio src="audio.mp3" controls></audio>

<script>
const audio = document.querySelector('audio');
audio.addEventListener('progress', () => {
  const duration = audio.duration;
  const buffered = audio.buffered.end(audio.buffered.length - 1);
  const percent = (buffered / duration) * 100;
  console.log(`Download progress: ${percent.toFixed(2)}%`);
});
</script>

在这个例子中，我们将一个名为“audio.mp3”的音频文件作为<audio>标签的源文件，然后使用addEventListener()方法监听progress事件。每次事件触发时，我们都计算音频文件的下载进度并打印出来。要计算下载进度，我们使用audio.duration属性获取音频文件的总持续时间，然后使用audio.buffered属性获取已下载部分的范围，并计算已下载部分占总持续时间的百分比。

需要注意的是，由于progress事件只在音频文件下载时触发，因此它的触发频率取决于用户的网络速度以及音频文件的大小。如果您想在音频文件下载完成后执行操作，可以使用canplaythrough事件。

# ratechange 音频的播放速率发生变化。

ratechange是HTML5 <audio>标签的一个事件，它在音频播放速率变化时触发。用户可以通过在浏览器的播放器控件中调整播放速率，或者使用playbackRate属性以编程方式调整播放速率。

下面是一个简单的示例，演示了如何监听ratechange事件：

<audio src="audio.mp3" controls></audio>

<script>
const audio = document.querySelector('audio');
audio.addEventListener('ratechange', () => {
  console.log(`Playback rate changed to ${audio.playbackRate}`);
});
</script>

在这个例子中，我们将一个名为“audio.mp3”的音频文件作为<audio>标签的源文件，并使用addEventListener()方法监听ratechange事件。每次事件触发时，我们都打印出音频的当前播放速率。

要以编程方式设置音频的播放速率，可以使用playbackRate属性，例如：

audio.playbackRate = 1.5; // 将播放速率设置为1.5倍

需要注意的是，不是所有浏览器都支持ratechange事件，因此您应该在使用它之前检查浏览器的兼容性。

下面是一个简单的示例，演示了如何监听seeked事件：

<audio src="audio.mp3" controls></audio>

<script>
const audio = document.querySelector('audio');

要解决stalled事件，可以尝试以下方法：

检查网络连接并确保它正常工作。

# timeupdate 音频的播放位置发生变化。

timeupdate是HTML5 <audio>标签的一个事件，它在媒体播放器的当前时间被更新时触发。这个事件通常用于创建自定义的音频播放器控制，例如进度条和播放时间显示。

下面是一个简单的示例，演示了如何监听timeupdate事件并更新一个进度条和播放时间显示：

<audio src="audio.mp3" controls></audio>
<div>
  <progress value="0" max="100"></progress>
  <span id="current-time">0:00</span>
</div>

<script>
const audio = document.querySelector('audio');
const progress = document.querySelector('progress');
const currentTime = document.querySelector('#current-time');

audio.addEventListener('timeupdate', () => {
  const { currentTime: currentTimeValue, duration } = audio;
  const progressPercentage = (currentTimeValue / duration) * 100;
  progress.value = progressPercentage;
  currentTime.textContent = formatTime(currentTimeValue);
});

function formatTime(time) {
  const minutes = Math.floor(time / 60);
  const seconds = Math.floor(time % 60);
  return `${minutes}:${seconds.toString().padStart(2, '0')}`;
}
</script>

在这个例子中，我们使用<audio>标签加载一个名为“audio.mp3”的音频文件，并创建一个包含一个进度条和当前播放时间的<div>元素。然后，我们使用addEventListener()方法来监听timeupdate事件，并在事件处理程序中更新进度条和播放时间显示。我们还定义了一个名为formatTime()的辅助函数，用于格式化播放时间的显示。

要注意的是，timeupdate事件可能会频繁触发，具体取决于浏览器和媒体文件的性能。在处理这个事件时，我们应该注意避免使用过多的计算和DOM操作，以避免影响性能。

# volumechange 音频的音量发生变化。

<audio> 标签的 volumechange 事件会在音量改变时触发，允许开发者监听音量的变化并作出相应的操作。

该事件可以通过 JavaScript 监听，方法如下：

const audio = document.querySelector('audio');

audio.addEventListener('volumechange', function() {
  console.log('Volume changed to ' + this.volume);
});

在这个示例中，当音量发生变化时，事件侦听器将被调用并将音量打印到控制台上。

除了 JavaScript 监听器外，还可以使用 onvolumechange 属性将处理程序直接附加到元素上，如下所示：

<audio controls onvolumechange="handleVolumeChange()">
  <source src="audio.mp3" type="audio/mpeg">
</audio>

在这个示例中，当音量发生变化时，handleVolumeChange 函数将被调用。

请注意，该事件不仅在通过音量控件调整音量时触发，而且在通过编程方式调用 volume 属性或调用 muted 属性时也会触发。

<audio controls onwaiting="handleWaiting()">
  <source src="audio.mp3" type="audio/mpeg">
</audio>

在这个示例中，当播放器因缓冲不足而暂停播放时，handleWaiting 函数将被调用。
